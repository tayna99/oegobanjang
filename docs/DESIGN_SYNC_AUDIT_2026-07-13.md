## 0. 결론 요약

1. **온보딩·CSV 목업 ↔ Montage DS 정합: 우수.** 두 파일의 hex·rgba 값 전량이 토큰 램프 값과
   일치하거나(§1), 근접값(§1-1건)만 예외. 아웃라인은 실제 화면 요소에서 전부 `inset box-shadow`이고,
   `border:1px solid` 사용은 전부 목업 캔버스의 카드/폰 프레임 장식(구현 대상 아님)이다.
2. **마스킹 형식 불일치 확인(기지 사항, §2).** 두 목업 모두 외국인등록번호를 `900412-6●●●●●●`
   부분 마스킹으로 표기한다 — 코드리뷰로 강화된 `lib/mask.ts`(전체 마스킹, 숫자 0개 규칙,
   `PackagePage.test.tsx:44`)와 충돌. **사용자 결정: 전체 마스킹 유지** — 목업의 부분 마스킹은
   "디자인 내부 결함"으로 기록하고 구현에 반영하지 않는다.
3. **PC 재수입은 부분 캡처.** `DesignSync get_file`이 256KiB에서 잘라(`truncated:true`) 4a~4f
   구간만 온전히 확보했다 — 3a~3c/2a~2d/v1 구간은 기존 frozen 파일이 계속 근거다(§3-0).
4. **PC 4a~4f 델타 분류(§3):** 4e는 기 구현(Phase D), 4a/4c/4f는 기존 화면의 부분 확장,
   4b/4d는 순신규. 이번 작업 범위(사용자 결정): **PC 순신규 화면까지 전부 포함.**

- **아웃라인 = inset box-shadow** 규칙 준수: 실제 화면 요소 41회(온보딩)/10회(CSV) 전부 `inset 0 0 0 …px`. `border:1px solid` 각 7/8회는 전부 목업 카드·폰 프레임 장식(`외고반장 PC.dc.html` 감사에서도 동일 패턴 확인됨) — 구현 대상 아님.
- 가드레일 고정 문구 확인: "승인 전에는 외부 발송이 차단됩니다."(온보딩 O5) · "가능/불가능 판단은 제공하지 않습니다."(CSV) 원문 그대로 존재.
- 타이포: Pretendard, 이모지·느낌표 UI 0건.

## 3. PC 재수입 — 4a~4f 델타 분류

**§3-0 캡처 한계:** 원격 `외고반장 PC.dc.html`은 256KiB 조회 한계를 넘겨 `truncated:true`로 응답했다.
4a~4f(약 90KB, id="4a"~id="4f", 3a 시작 직전까지)만 온전히 캡처해 `외고반장 PC_4a-4f(신규티어).dc.html`로
저장했다. 3a~3c/2a~2d/v1 구간은 캡처하지 못했으나, 새 파일의 3a 섹션 제목("PC 통합 재설계 — 컨트롤
타워 · 워크벤치 · 거버넌스")이 기존 frozen 파일의 §3a 제목과 정확히 일치해 **미변경으로 판단**한다
(바이트 단위 대조는 아님 — 제목·섹션 헤더 텍스트 대조).

**색상 재확인(4a~4f 구간, 온전 캡처):** `#171719`/`#0066FF`/`rgba(55,56,60,.61)`/severity 페어 전부
기존 값과 동일 — 새 색상 발명 없음. 4e에 `900412-6●●●●●●` 재등장(§2 결함과 동일 판정).
