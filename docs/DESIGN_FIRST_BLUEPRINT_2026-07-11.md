# Design-First 블루프린트 — 디자인 소스에 코드베이스를 맞추는 설계 (2026-07-11)

> 지시(2026-07-11): "디자인 소스에 코드베이스를 맞춰줘. 코드베이스에 맞추지 말고 … 설계를 한 후 그에 필요한 코드를 작성하는 식으로."
> 소스: `reference/design-system/` 고정본 3종 — `외고반장 PC.dc.html`(통합 재설계 3a·3b·3c 채택), `외고반장 Mobile.dc.html`(승인 큐 중심 개편 2a~2d — **이 지시로 기존 '보류'를 뒤집고 채택**), `Montage 공용 컴포넌트.dc.html`(자체 컴포넌트 6종 스펙).
> 근거 분석: 스펙 md 9종 전수 digest + Mobile 전문 추출 + PC 3a/3c/관제형 2d 전문 추출(2026-07-11, 세션 기록). 선행 검수: `docs/DESIGN_SYNC_AUDIT_2026-07-11.md`.

## 0. 설계 원칙 (이 문서의 모든 결정이 따르는 것)

1. **디자인이 그린 화면은 디자인이 정답이다.** 코드·데이터 모델·목업 데이터·규칙 문서가 디자인과 다르면 **코드와 문서를 고친다**(반대 방향 금지). 스펙 문서와 디자인이 충돌하면 디자인을 채택하고, 해당 스펙 조항은 "개정됨(superseded)"으로 명시 기록한다.
2. **디자인이 안 그린 화면은 스펙 트랙을 유지한다.** 전수 검색으로 확인된 미포함 표면: M6 응답 해석, M8 전역 판단 기록, M9 에이전트 런·재생, 커맨드바(D3), 자율성 사다리(9단계 P1-6), 온보딩 O1~O5, CSV 업로드. 이들은 1단계·통합설계 스펙이 계속 소스다. 디자인에 없다는 이유로 지우지 않는다.
3. **가드레일은 디자인보다 위에 있다 — 단, 디자인 자신도 그 가드레일을 선언하고 있다.** 디자인 내부에서 자기 규칙(고정 문구·고위험 처리·기록 색 규칙)과 어긋난 3건은 "디자인 제작 오류"로 판정하고 정본대로 교정해 구현한다(§2).
4. **한 세계관.** Mobile 2a~2d와 PC 3a~3c는 같은 6인 로스터·같은 파이프라인 어휘·같은 판단 기록 번호대(#4783~#4791)를 쓴다. 데이터 모델 확장은 한 번에 설계해 두 표면이 공유한다(§3).

## 2. 가드레일 판정 (디자인 자기모순 교정 + 규칙 개정)

**교정(디자인 제작 오류 — 정본대로 구현):**
- **C1** Mobile §2c 배너 제목 "승인 전 외부 발송이 차단됩니다"(159행) → 정본 **"승인 전에는 외부 발송이 차단됩니다."** 로 교정(GOTCHAS "글자 하나도 바꾸지 않는다"; 같은 파일 85행·SafetyNotice가 정본). 배너 본문("승인 완료 시 발송 실행이 가능해지며, 승인자는 판단 기록에 남습니다.")은 채택.
- **C9** Mobile §2d 타임라인 노드 색 → 탭별기획 §4.2 정본(사람 결정 노드만 primary, 시스템 노드 무채색)으로 교정. 구조·6노드·해시 표시는 디자인대로.
- **C10** PC §3a 고위험(기한 경과) 행의 일반 "승인" 버튼 → 같은 케이스를 Mobile §2a가 "검토"로 처리하는 것에 맞춰 **행 액션을 "검토"(중립)로 교정**, 승인 진입은 워크벤치/체크리스트에서 행정사 전달 승인으로만(GOTCHAS "high risk = 전달 전용, 처리 버튼 금지").

**비준(디자인이 새로 도입한 규칙 — 채택하고 문서화):**
- **C2** §3a 각주 "승인은 케이스 단위로만 가능합니다." → **비준.** 일괄 승인 금지 규칙으로 GOTCHAS에 추가.
- **C11** 근거 등급 **F(합성 데이터 — 근거 사용 불가)** → **비준.** `CitationGrade`에 'F' 추가(C는 존치, 미사용).

**개정(스펙 조항이 디자인에 의해 대체됨 — superseded 기록):**
- 카드 CTA 정확히 2개(GOTCHAS §3, 1단계 §0.2) → **개정: 브리핑 카드 CTA는 "검토" 1개.** 승인 진입은 검토 깔때기(2b→2c)로만.
- M2 = 바텀시트(1단계 §M2, ARCHITECTURE §3) → **개정: 모바일 케이스 상세는 전면 페이지.**
- M4 = 스트리밍 게이트(1단계 §M4, GOTCHAS §2) → **개정: 승인 게이트는 사람 체크리스트 4/4 + 근거≥1(citation-0 잠금은 유지).** 스트리밍 런은 M9 트랙(재생/커맨드)으로 이동.
- **데모 대본(8단계) 영향 — 사용자 인지 필요:** 3막의 "스트리밍이 끝나기 전에는 승인 버튼이 눌리지 않습니다" 와우포인트가 "필수 체크리스트를 끝내기 전에는 승인되지 않습니다"로 바뀌고, 2막의 "런 #4788 보기" 재생은 2b의 판단 기록 링크(#4788)를 통해 `/run/4788` 재생 뷰로 연결해 유지한다. 4막(커맨드바 런)은 커맨드바 존치로 유지.
