## 2. Semantic orchestrator v2 후속 수정

merge 이후 로컬에서는 `backend/app/services/agent_chat_rag.py`를 더 확장했다. (초기 RAG-first가 "검색 + intent 선택"에 가까웠다면, 후속 수정은 "semantic retrieve -> LLM normalize -> tool execute -> LLM grounded answer"의 4단계 orchestration으로 정리하는 작업이다.)

추가/수정한 흐름:

- `ORCHESTRATION_VERSION = "semantic_rag_llm_tools_v2"` 추가
- tool trace 이름을 `agent_chat_rag_search`에서 `agent_chat_semantic_retrieve` 중심으로 변경
- `agent_chat_llm_normalize`, `agent_chat_tool_execute`, `agent_chat_llm_grounded_answer` 단계를 응답에 노출
- `normalized_intent`, `normalized_entities`, `executed_tools` 필드 추가
- OpenAI뿐 아니라 Ollama 호환 provider 설정도 추가
- forbidden action을 단순 문자열이 아니라 blocked action category로 분리

blocked action category 예:

```txt
send_message_without_approval
government_portal_submission
complete_status_without_approval
discriminatory_or_attrition_prediction
```

이 변경의 목적은 "사용자가 자연스럽게 말한 질문"을 바로 LLM 답변으로 흘려보내는 것이 아니라, 먼저 업무 intent와 실행 가능한 도구 단계를 정규화하고, 승인 필요한 작업은 실행하지 않도록 만드는 것이다. (외고반장의 제품 원칙상 메시지 발송, 정부 제출, 차별적 추천은 자동 실행되면 안 된다.)

## 6. 오늘 확인한 품질 이슈

다국어 챗봇에서 오늘 확인한 핵심 이슈는 세 가지다.

첫째, intent 감지는 잘 된다. (다국어 서류 요청 문장 대부분이 `document_request_message`로 들어간다.)

둘째, guardrail도 작동한다. (카톡 즉시 발송 요청은 `unsupported`로 막힌다.)

셋째, 실제 언어 생성 품질은 아직 부족하다. (베트남어/태국어 요청인데 한국어 문구가 나오는 경우가 있고, 답장 원문 없이 답장 요약을 만들어낸다.)

따라서 후속 수정은 아래가 우선이다.

- `document_request_message`에서 target language를 `normalized_entities`로 확실히 뽑기
- grounded answer prompt에 "요청 언어로 초안 작성"을 강제하기
- `worker_reply_summary` 또는 `reply_interpretation` intent를 별도로 만들기
- 답장 원문이 없으면 요약하지 않고 원문 입력을 요청하기
- 즉시 발송 요청은 계속 `send_message_without_approval`로 막기
