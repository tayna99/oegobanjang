
# Security Guardrails

## 1. 목적

외고반장은 외국인 근로자의 개인정보, 체류 정보, 고용 정보, 서류 상태를 다룬다.  
따라서 보안과 법적 책임 분리는 기능 구현보다 우선한다.

---

## 2. 개인정보 취급 원칙

다음 정보는 민감하게 취급한다.

- 이름
- 생년월일
- 국적
- 여권번호
- 외국인등록번호
- 체류자격
- 체류만료일
- 계약 정보
- 숙소 정보
- 연락처
- 제출 서류

원칙:

- 필요한 정보만 저장한다.
- 로그에는 민감정보 원문을 남기지 않는다.
- 외국인등록번호와 여권번호는 마스킹한다.
- 문서 export 시 담당자 승인 후 처리한다.
- 외부 발송 전 반드시 Human Approval을 거친다.

---

## 3. 로그 마스킹 규칙

```txt
외국인등록번호: 900101-1234567 → 900101-*******
여권번호: M12345678 → M1234****
전화번호: 010-1234-5678 → 010-****-5678
```

Evidence Log에는 원문 대신 다음 정보만 저장한다.

- 문서 존재 여부
- 서류 종류
- 판단 근거 source_id
- 승인자 ID
- 처리 시각
- 작업 결과 요약

---

## 4. 법적 책임 분리

AI는 다음을 하지 않는다.

- 비자 발급 가능 여부 확정
- 체류 연장 가능 여부 확정
- 법률 자문
- 노무 자문
- 정부 포털 제출
- 행정사/노무사 자동 전달
- 대외 문서 자동 발송

대신 다음처럼 응답한다.

```txt
공식 근거와 현재 입력된 정보를 기준으로 검토가 필요한 항목을 정리했습니다.
최종 판단과 제출은 담당자 또는 행정사 검토가 필요합니다.
```

---

## 5. 감시·차별 금지

외고반장은 외국인 근로자를 감시하거나 평가하는 도구가 아니다.

금지:

- SNS 감시
- 단톡방 감시
- 국적 커뮤니티 추적
- 이탈 가능성 예측
- 성실도 점수화
- 국적별 선호
- 사업장 공개 평판 점수
- 브로커 색출

허용:

- 자발적 상담 내용 요약
- 제출 서류 상태 정리
- 지원 필요 신호 정리
- 공식 상담 채널 안내

---

## 6. 승인 필요 작업

다음 작업은 초안 생성만 가능하며, 실제 실행은 담당자 승인 후 가능하다.

- 외국인 근로자에게 메시지 발송
- 행정사/노무사에게 패키지 전달
- 케이스 상태 완료 처리
- PDF export
- 카톡/문자 푸시 발송

---

## 7. 금지 응답 예시

```txt
이 근로자는 비자 연장이 가능합니다.
이 후보자는 성실해 보입니다.
이 국적 근로자를 우선 추천합니다.
바로 정부 포털에 제출하겠습니다.
근로자의 SNS를 확인해 이탈 가능성을 분석했습니다.
```

```txt
최종 비자 가능 여부는 행정사 검토가 필요합니다.
```

```txt

```
