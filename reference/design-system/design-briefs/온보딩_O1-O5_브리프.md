# 디자인 브리프 — 온보딩 O1~O5 (Claude Design용)

> 용도: 이 문서를 claude.ai/design "Mobile screen design" 프로젝트에 그대로 붙여 목업을 생성한다.
> 생성된 `.dc.html`은 `reference/design-system/`에 고정 후 ROADMAP 4.1로 구현한다(블루프린트 §9-B).
> 근거 스펙: `reference/specs/3단계_온보딩플로우_v1.md`, `reference/specs/7단계_권한모델_승인위임_v1.md`.

## 디자인 시스템 (반드시 준수)

- **Montage(Wanted) v2** 토큰만 사용: primary `#0066FF`, label-normal `#171719`, label-alternative `rgba(55,56,60,.61)`, bg `#FFFFFF`/`#F7F7F8`, 라인 `#E1E2E4`. severity 페어(critical `#E52222`/`#FEECEC` 등)는 `colors_and_type.css` 램프에서.
- 폰트 Pretendard. 타입 스케일: heading1 22 / heading2 20 / body1 16 / body2 15 / label1 14 / caption1 12.
- 버튼 라디우스 8/10/12, 아웃라인은 CSS border 대신 inset box-shadow. 이모지·느낌표·마스코트 금지.
- 모바일 375px 프레임. 하단 고정 CTA(h48).
- 이미 만든 컴포넌트 재사용: BackHeader(뒤로가기 앱바), Button, 입력은 inset-shadow 박스, BottomSheet, SafetyNotice.

## 화면 5종 (플로우: O1→O2→O3→O4→O5)

### O1 — 전화번호 인증 + 승인용 본인확인 설정
- 큰 제목 "시작하기" + 서브라인 "휴대폰 번호로 시작합니다".
- 전화번호 입력(국가코드 +82 고정) + "인증번호 받기" 버튼.
- 인증번호 6자리 입력(6칸 세그먼트).
- 하단 안내: 생체/PIN을 **승인 시 본인확인**에 쓴다는 1줄(7단계 §4). 실제 생체 등록은 목업(스위치 1개).
- 고정 문구 없음. CTA "다음".

### O2 — 역할 선택 (대표 / 담당자)
- 제목 "어떤 역할로 시작하시나요?".
- 카드 2개(선택형): **대표(owner)** — "승인 권한·자율성 승급 결정", **담당자(manager)** — "케이스 검토·승인 요청 처리". 선택 시 primary 아웃라인.
- 역할별 권한 차이 1줄 요약(7단계 §2). CTA "다음".

### O3 — 사업장 4필드 입력
- 제목 "사업장 정보".
- 4필드: 사업장명 / 지역 / 업종 / E-9 근로자 수. (3단계 O3)
- 각 필드 label1 + inset-shadow 입력. CTA "다음".

### O4 — 첫 근로자 등록 (3경로 선택)
- 제목 "첫 근로자를 등록하세요" + 서브라인 "1명·필드 4개면 시작할 수 있습니다".
- 3경로 세그먼트(디자인 D6): **서류 사진(OCR)** / **직접 입력** / **CSV 업로드(PC 권장)**.
  - 직접 입력 선택 시: 이름·국적·팀·체류만료일 4필드.
  - **외국인등록번호는 마스킹 저장**(3단계 O4, `900412-6●●●●●●` 형태 표기) — 원문 미표시.
- 하단 SafetyNotice(neutral): "승인 전에는 외부 발송이 차단됩니다."
- CTA "등록하고 브리핑 만들기".

### O5 — 첫 브리핑 동기 생성 (Aha)
- **로딩 연출**: "6명의 체류·계약·서류를 확인하는 중…" + 진행 스텝(감지→근거 수집→초안) shimmer/StepTimeline 세로형.
- 완료 시: "첫 브리핑이 준비됐어요" + 첫 카드 1장 프리뷰(항상 비어 있지 않게 — 3단계 O5 "always-non-empty").
- CTA "오늘 브리핑 보기" → M1.
- 진행 중 절대 "자동으로 처리합니다" 류 표현 금지 — "확인하고, 승인 후 진행합니다".

## 가드레일 (디자인에 반드시 반영)
- 외국인등록번호·여권번호 원문 표기 금지(마스킹만).
- 단정형("가능합니다") 금지, 이모지·느낌표 금지.
- O5 로딩은 "준비/확인" 어휘만. 발송·제출 표현 금지.
