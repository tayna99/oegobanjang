> 근거 스펙: `docs/MESSAGING_CHANNELS.md` §3(수신 파이프라인)·§5 ②, `reference/specs/2단계_알림카탈로그_딥링크맵_v1.md` §4.1(N02=worker_replied).

## 디자인 시스템 (반드시 준수)

- **Montage(Wanted) v2** 토큰만: primary `#0066FF`, 본문 `#171719`, 보조 텍스트
  `rgba(55,56,60,.88)`, 3차 텍스트 `rgba(55,56,60,.61)`, 배경 `#FFFFFF`/`#F7F7F8`.
  "승인 필요(정보)" 톤(파란 배경 카드 — 만료/완료 안내에 사용) `#EAF2FE` bg / `#0066FF` text.
- 폰트 Pretendard. 타입: heading2 20 / body1 16 / body2 15 / label1 14 / caption1 12.
- 버튼 라디우스 8/10/12, 아웃라인은 `border` 대신 inset box-shadow. 이모지·느낌표·마스코트 금지.
- **모바일 375px 프레임 1벌이면 충분** — 이 화면은 단일 반응형(PC 브라우저로 열려도 같은
  레이아웃)이라 별도 PC 캔버스 불필요. `ExpertLinkPage`와 동일하게 `max-w-screen-md
  mx-auto` 중앙 카드 컨테이너, 상단 Shell 탭바·네비 없음(무인증 단독 페이지 — 진입 즉시
  본문).
- 이미 만든 컴포넌트 재사용: Button(하단 고정 CTA h48), 안내 카드(`ExpertLinkPage`의 만료
  상태 박스 — 파란 톤 카드 패턴), SafetyNotice 없이 이 화면 전용 1줄 안내 텍스트만.

## 화면 구성 (5개 상태 — 전부 목업에 포함)

### 1. 정상 — 메시지 확인 + 응답
- 상단: 발신자 표시("담당자 메시지" 또는 회사명) + 수신 시각.
- 메시지 원문 카드(근로자 모국어, 예: 베트남어) — 배경 `#F7F7F8`, 라운드 카드.
- 응답 UI 2종:
  - **모국어 버튼형 선택지**(메시지 유형별 프리셋 3개 내외 — 예: "확인했습니다 / 다시
    보내주세요 / 질문이 있습니다"에 대응하는 모국어 문구). 세로 스택, outline 버튼.
  - **자유 텍스트 입력**(선택 사항, 아래로 갈수록 부차적임을 여백으로 표현) — placeholder
    "직접 입력하시려면 여기에 적어주세요"류.
- 하단 고정 CTA "보내기"(자유 텍스트 비어있고 버튼 선택도 없으면 disabled).
- 안내 1줄(상단 또는 하단): "담당자가 확인한 뒤 반영됩니다" — 즉시 상태 변경이 아님을
  명시(과대 약속 금지).

### 2. 제출 완료
- 체크 아이콘(SVG, 24×24 currentColor — 이모지 금지) + "응답이 접수됐습니다".
- 재제출 UI 없음(잠금) — `ExpertLinkPage`의 "회신 보내기" 후 잠금 관례와 동일.
- CTA 없음 또는 "닫기"만(브라우저 뒤로가기로도 충분하면 CTA 자체를 생략해도 됨 — 디자인
  결정 위임).

### 3. 만료됨
- `ExpertLinkPage`의 만료 상태 박스 **그대로 재사용**: 중앙 정렬, `max-w-sm` 카드,
  "링크가 만료되었습니다" + "보안을 위해 만료형 링크로만 전달됩니다. 담당자에게 재발급을
  요청해주세요"류 안내(문구는 근로자 대상으로 모국어 번역 버전 필요 — 한국어 1벌 + 번역
  여부는 디자인/구현 단계에서 결정).

### 4. 이미 응답함
- "이미 응답을 보내셨습니다" + 제출 시각. 이전 응답 원문은 재노출하지 않는다(가드레일 참고).

### 5. 링크 없음/무효
- `ExpertLinkPage`의 "링크를 찾을 수 없습니다" **그대로 재사용** — 화면 중앙 텍스트 1줄뿐,
  다른 정보 없음(존재 자체 비노출).

## 가드레일 (디자인에 반드시 반영)

- 근로자 응답 원문은 이 페이지(제출 시점)와 스레드 내부에서만 노출 — 목록 미리보기·Evidence
  요약에는 절대 포함하지 않는다(GOTCHAS §3). "이미 응답함" 상태에서도 이전 응답 원문을
  다시 보여주지 않는다.
- 이 페이지의 제출이 케이스 상태를 직접 바꾸지 않는다 — M6 해석 큐로만 들어가고 담당자
  확인 후 반영된다(`isFinal:false` 계약). "담당자가 확인한 뒤 반영됩니다"류 문구 필수.
- 외국인등록번호 등 PII 표기 전면 금지(이 화면엔 애초에 등장할 필드가 없어야 함).
- 단정형 표현("가능합니다" 오용) · 이모지 · 느낌표 금지.
- "링크 없음"과 "만료됨"을 구분해 보여주되, 둘 다 존재 여부 자체는 비노출(만료는 "한때
  존재했음"을 알려도 되는 경우이나 링크 없음은 절대 그 이상 정보를 주지 않는다).
