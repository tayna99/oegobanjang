# 디자인 브리프 — 알림 설정 화면 (설정 > 알림, Claude Design용)

> 용도: claude.ai/design "Mobile screen design" 프로젝트에 붙여 목업 생성 →
> `reference/design-system/` 고정 → ROADMAP R5.4(알림·푸시) 선행 조각으로 구현.
> 근거 스펙: `reference/specs/2단계_알림카탈로그_딥링크맵_v1.md` §6(알림 설정 화면
> 요구사항 — 표까지 이미 확정돼 있으나 화면 자체가 없던 항목).
> 기존 코드 참고: `src/features/settings/SettingsHubPage.tsx` — 이 화면과 같은 크롬(행
> 목록 `SettingsRow` 관용구, 승인 정책 섹션의 세그먼트 버튼 그룹)을 그대로 재사용한다.
> 새 최상위 화면이 아니라 **설정 허브에서 진입하는 신규 서브페이지**다.

## 디자인 시스템 (반드시 준수)

- **Montage(Wanted) v2** 토큰만: primary `#0066FF`, 본문 `#171719`, 보조 텍스트
  `rgba(55,56,60,.88)`/`.61`, 배경 `#FFFFFF`/`#F7F7F8`, 라인 `rgba(112,115,124,.08~.22)`.
- 타입: heading2 20(타이틀) / label1 14(행 라벨) / caption1 12(부제·경고 문구).
- **단일 반응형** — `SettingsHubPage`·`MembersPage`·`DelegationPage`와 동일하게
  `useIsDesktop` 분기 없이 한 레이아웃(모바일 캔버스 1벌로 제출, PC에서도 같은 화면 그대로
  동작).
- 이미 만든 컴포넌트 재사용: `BackHeader`(뒤로가기 앱바), `SettingsRow`(행 목록 —
  라벨+부제+`›`, `border-b border-hairline`), 승인 정책 섹션의 세그먼트 버튼 그룹
  (`role="group"`, 선택 시 `bg-approvalbg`/`text-approval`+`shadow-rail-focus`).

### 미해결 — 토글(ON/OFF) 표현 방식 (디자인이 결정할 것)

이 코드베이스에는 **아직 토글/스위치 컴포넌트가 없다**(전례는 2택 세그먼트 버튼 그룹뿐 —
"승인 정책" 섹션). 이 화면은 6개 항목 중 다수가 진짜 ON/OFF(끄고 켜기)라 세그먼트 버튼
그룹을 그대로 쓰면 어색할 수 있다. 아래 두 안 중 하나를 디자인 단계에서 확정한다:
- **(A) 신규 토글 스위치 컴포넌트**를 만든다 — Montage 토큰(라인·primary) 안에서 작은
  pill 스위치. 새 컴포넌트가 생기므로 `rules/design.md`에 스펙 추가 필요.
- **(B) 기존 세그먼트 버튼 그룹을 ON/OFF 2택으로 재사용**한다 — 새 컴포넌트 없이 기존
  관용구만으로 완결. `docs/GOTCHAS.md` §4 "같은 목적의 컴포넌트를 두 번 만들지 않는다"
  원칙에는 (B)가 더 부합.
목업 제출 시 어느 쪽을 택했는지 명시할 것 — 자가 검수 체크리스트에도 포함.

## 화면 구성 (모바일 375px, 설정 허브 → "알림" 행 → 이 화면)

### 상단
- `BackHeader` "알림"(뒤로가기 → 설정 허브).

### 섹션 1 — 브리핑
- 행: "아침 브리핑 시각" — 부제 "회사 단위 설정(개인 아님)" + 현재 값(예: 08:30) + `›`
  또는 인라인 time picker(기존 `SettingsHubPage`의 브리핑 시각 입력 패턴 재사용 —
  `<input type="time">` 인라인 스타일).

### 섹션 2 — 즉시 알림
| 항목 | 기본값 | 표현 |
|---|---|---|
| 승인 요청 즉시 알림 | ON | **토글 비활성(항상 ON, 끌 수 없음)** — 비활성 사유 1줄 캡션 "승인 루프는 끌 수 없습니다"류. 승인 권한자(owner/manager)에게만 이 행 노출, viewer는 섹션 자체 비노출. |
| 응답 도착 즉시 알림 | ON | 토글 가능. OFF 시 캡션 "OFF 시 아침 다이제스트로만 받습니다"류로 즉시 갱신. |
| CRITICAL 야간 알림 | ON | 토글 가능하되 OFF 시 경고 캡션(오렌지 톤 아님 — 일반 경고 문구, 색상 강조 없이 텍스트로만) "야간 CRITICAL 알림을 끄면 긴급 사안을 늦게 확인할 수 있습니다"류. |

### 섹션 3 — 요약
| 항목 | 기본값 | 표현 |
|---|---|---|
| 주간 요약 | OFF | 옵트인 토글. |

### 섹션 4 — 채널 (읽기 전용)
| 항목 | 값 | 표현 |
|---|---|---|
| 채널 우선순위 | 푸시 우선 | **읽기 전용 행**(`›` 없음, 클릭 불가) — "알림톡 폴백" 여부를 부제로 표시. |

## 가드레일 (디자인에 반드시 반영)

- "승인 요청 즉시 알림"은 시각적으로도 명확히 "끌 수 없는 항목"으로 읽혀야 한다(회색조
  비활성 토글 + 사유 캡션 — 사용자가 껐다고 착각하게 만드는 표현 금지).
- 채널 우선순위 행이 실제로는 조작 불가(읽기 전용)임을 시각적으로 구분(`›` 화살표 없음,
  다른 행과 다른 톤).
- viewer 역할은 이 화면 진입 시 `SettingsHubPage`의 기존 관례("열람자 권한으로는 설정에
  진입할 수 없습니다")를 그대로 따른다 — 별도 예외 만들지 않는다.
- 단정형·이모지·느낌표 금지. 경고 캡션도 "위험합니다"류 과장 없이 사실 서술로만.
