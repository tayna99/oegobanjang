## 0. 데모의 한 문장

> "아침에 알림 하나가 오고, 사장님이 출근길에 3번 탭하면, 놓치면 사고가 될 일이 승인·기록까지 끝나 있습니다."

기존 데모2와의 차이: 시작점이 "담당자가 PC를 연다"에서 **"시스템이 먼저 말을 건다"**로 바뀐다. Zero-Input First를 서사 자체로 증명하는 구성.

## 2. 본편 — 4막 4분

### 1막. 알림이 먼저 온다 (30초)

```
[연출] 잠금화면 목업 1장 (슬라이드)
  푸시: "오늘 브리핑 — 승인 필요 2건 · 우선 확인 1건"       ← 2단계 N10
  알림톡 버전도 나란히: 앱 없는 사장님에게는 카카오로 갑니다
[멘트] "사용자는 아무것도 입력하지 않았습니다.
        매일 아침 8시 30분, 룰 엔진이 6명의 체류·계약·서류를 계산해서 먼저 알려줍니다."
[탭] 알림 → M1 오늘 브리핑 진입 (딥링크)
```

### 2막. 브리핑 → 케이스 → 초안 (90초)

```
[M1] "담당자님, 오늘 확인이 필요한 외국인 고용 업무가 3건 있습니다."
      Nguyen 카드: D-30 · 누락 2건 · 승인 필요 + 추천 이유 1줄
      + 프로액티브 행: "AI가 준비를 마쳤습니다 · 런 #4788 보기"
[탭] 런 #4788 보기 → 재생 뷰(읽기 전용): 오늘 07:58 D-30 감지로 자동 실행 —
      서류 점검 → 규정 검색 → 에이전트 핸드오프 → 초안 생성 → 가드레일 정지(승인 요청 생성)
[멘트] "사용자가 앱을 열기 전인 아침 7시 58분, 에이전트가 스스로 이 카드를 준비했습니다.
        무엇을 확인했는지 전 과정이 재생되고, 발송 직전에 스스로 멈춘 것까지 기록돼 있습니다.
        왜 이 카드가 1번인지 — 추천 이유가 항상 붙습니다. AI의 판단은 설명 가능해야 하니까요."
[탭] 카드 → M2 바텀시트
      AI가 확인한 내용 / 누락 서류 체크리스트 / 근거 2건 (출입국관리법 제25조 · A등급)
[멘트] "모든 판단에는 법령 근거가 달립니다. 근거가 없으면 승인 버튼이 잠깁니다."
[탭] 초안 보기 → M3
      한국어/베트남어 토글, 개인정보 사용 목적·기한 자동 검사 통과 표시
      예상 응답 시나리오 3종 (긍정/추가 질문/지연 → 리마인드 예정)
[멘트] "사장님이 한국어로 확인하면, 근로자에게는 베트남어로 나갑니다.
        단, 아직 아무것도 발송되지 않았습니다 — 화면의 고정 문구를 봐주세요:
        '승인 전에는 외부 발송이 차단됩니다.'"
```

### 3막. Live Agent → 승인 (60초) — 와우 포인트

```
[탭] 보내기 승인 → M4
      단계 스트리밍: 프로필 확인 → 이전 요청 이력 → 초안 생성 → 발송 전 승인 대기
      "현재 자율성: Medium (승인 필요)" · 판단 기록 #4789
[멘트] "AI가 방금 무엇을 확인했는지 과정이 그대로 보입니다.
        그리고 스트리밍이 끝나기 전에는 승인 버튼이 눌리지 않습니다 — 성급한 승인 방지입니다."
[탭] 최종 승인 → M5 (push-in 모션)
      상태 변화: 승인 전 → 발송 승인 완료 · 판단 기록 #4789 저장
      다음 할 일: "2일 내 응답 없으면 리마인드 제안 (예정)"
[탭] 브리핑으로 돌아가기 → M1
      Nguyen 카드가 '승인 완료 · 응답 대기'로 바뀌어 하단 이동, 승인 대기 2→1
[멘트] "승인이 실제로 상태를 바꿨습니다. 화면만 바뀐 게 아니라,
        누가·언제·무슨 근거로 승인했는지가 감사 기록으로 남았습니다."
```

### 4막. 에이전트 런 — 자연어로 부리기 (60초)

```
[입력] M1 하단 커맨드 바: "이번 달 급한 직원만 정리해줘"
[M9]  스트리밍: 요청 분석 → 근로자 상태 조회(12명) → D-day 계산(2명)
       → 누락 서류 점검 → ⚠ 가드레일: "외부 발송은 승인 전 차단되어 승인 요청으로 전환했습니다"
       → 규정 검색(근거 A) → 결과: Nguyen·Tran 카드 + 판단 기록 #4790
[멘트] "챗봇이 아닙니다. LLM이 스스로 도구를 골라 호출하는 에이전트 루프입니다.
        자유는 루프에 있고, 경계는 도구에 있습니다 —
        발송 도구는 아예 승인 대기만 반환하도록 만들어져 있습니다.
        가드레일이 작동한 것도 숨기지 않고 단계로 보여줍니다."
[클로징] "버튼을 누르는 SaaS가 아니라, 목적을 말하면 경로를 짜는 Agentic OS입니다."
```

## 3. 확장 데모 (+2분, 질의 대응용)

| 확장 | 내용 | 근거 문서 |
|---|---|---|
| E1. 응답 도착 | (슬라이드) N02 푸시 → M6: 베트남어 원문 → 한국어 요약 → 상태 업데이트 제안(담당자 확인 필요) → 새 추천 액션 | M6 스펙, 프로토타입 Claude_00 목업 |
| E2. 온보딩 | "이 브리핑이 나오기까지 얼마나 걸리나?" → 3단계 타임라인: 가입부터 첫 카드까지 3분, 근로자 1명·필드 4개면 시작 | 3단계 문서 |
| E3. 권한·위임 | "사장이 승인 안 하면?" → 48h 재알림 → 72h 위임자 이관 → 대리 승인 배지 · first-decision-wins | 7단계 문서 |
| E4. PC 확장 | "대량 관리도 되나?" → PC 캡처 1장: 업무 큐 테이블·CSV 업로드·행정사 패키지 (반응형 웹 단일) | 6단계, D4 |

## 4. 예상 질문 → 답변 스크립트

| 질문 | 답 |
|---|---|
| 실제로 발송되나? | MVP는 승인까지. 발송 연동(Zalo/알림톡)은 승인 기반 어댑터로 이후 단계. 그래서 지금도 "발송 승인 완료"이지 "발송 완료"가 아니다 |
| AI가 비자 판정을 하나? | 하지 않는다. 판정성 요청은 가드레일이 "행정사 검토 요청"으로 전환한다 (M9 error 스펙) |
| 근거는 어디서? | 법령·공식 안내 RAG(A/B등급). 근거 없으면 승인 잠금 |
| 기존 SaaS와 차이? | 사용자가 메뉴를 찾지 않는다. 알림→카드→승인 3탭. 나머지는 에이전트가 준비 |
| 데이터는 안전한가? | 외국인등록번호 원문 미표시·미저장(마스킹), 기록은 append-only, 회사 간 격리(tenant scope) |

## 5. 데모 체크리스트 (시연 전)

- 프로토타입 브라우저 미리 열기 + `prefers-reduced-motion` 꺼짐 확인 (스트리밍 연출 필요)
- 승인 1회 실행 후에는 새로고침으로 초기화 (approved 상태 리셋)
- 잠금화면 알림 목업 슬라이드 1장, PC 확장 캡처 1장 준비
- 발표 멘트에서 금지 표현 점검: "자동으로 처리합니다" 금지 → "준비하고, 승인 후 넘깁니다"
