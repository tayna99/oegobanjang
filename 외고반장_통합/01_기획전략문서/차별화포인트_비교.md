# 차별화 포인트

### **1. 정부 사이트가 제공하지 않는 것 (= 외고반장 영역):**

| 영역 | 정부 제공 | 외고반장 |
| --- | --- | --- |
| 신청 폼 제출 | ✅ EPS/하이코리아 | ❌ (안 함, 안전 원칙) |
| 법령 조회 | ✅ 국가법령정보센터 | ✅ RAG로 사장님 언어로 변환 |
| **체류만료 D-day 자동 알림** | ❌ | ✅ |
| **여러 직원 일괄 모니터링** | ❌ | ✅ |
| **누락 서류 자동 탐지** | ❌ | ✅ |
| **다국어 메시지 자동 생성** | ❌ | ✅ |
| **외국인 답변 한국어 요약** | ❌ | ✅ |
| **행정사 전달 패키지** | ❌ | ✅ |
| **카카오톡으로 들어옴** | ❌ | ✅ |

정부시스템의 UX는 사장님 페르소나에 안맞음

- 공인인증서 또는 간편인증 로그인
- 메뉴 5–7단계 깊이 들어가서 신청서 찾기
- 30+ 필드 수동 입력
- 첨부서류 따로 준비
    
    → 행정사 쓰는 이유
    
- 정부는 절대 카톡으로 알림 안 보내고, 절대 베트남어 메시지 안 만들어줌. 외고반장의 역할은 **그** 절차를 50–60대 한국 사장님이 베트남 직원 35명에게 적용할 수 있게 만드는 것
    
    ```jsx
    EPS는 사장님이 들어가서 신청해야 함.
    외고반장은 사장님이 카톡에 가만히 있어도 'Nguyen D-30 임박' 알림을 보냄.
    하이코리아는 체류 신청을 받음. 
    외고반장은 신청 전 누락 서류와 다국어 안내를 자동으로 준비함.
    정부 사이트는 한국어 단방향. 
    외고반장은 외국인 직원과의 다국어 양방향 소통을 처리함.
    ```
    
    #### 다국어 + 카카오톡 + 일괄 관리는 정부가 안 다룸
    
    **왜 정부 시스템만으로는 부족한가**
    
    | 정부 시스템 (EPS, 하이코리아, 정부24) | 현장 사장님이 실제로 겪는 일 |
    | --- | --- |
    | 신청 폼·법령·절차 제공 | 50명 직원 만료일 본인이 기억해야 함 |
    | 한국어 단방향 | 베트남어로 직원과 소통해야 함 |
    | 사장님이 직접 들어가서 입력 | 본업 하느라 사이트 들어갈 시간 없음 |
    | 누락 서류 검증 안 함 | 행정사한테 가서야 부족한 거 발견 |
    | 알림 없음 | 만료 지나서 알게 됨 → 행정 사고 |
    |  |  |

### 2. 워크비자가 실제로 하는 일 : 외국인 구인구직 매칭 마켓플레이스 + 비자 자가진단 툴

- **인재풀** (5,625명 DB) — 외국인 구직자 정보를 지역/국적/비자/한국어/학력으로 필터링
- **채용 공고** — 사업자가 올리는 잡 보드
- **비자 내비게이터** — 외국인이 자기 비자 자가진단하는 시뮬레이터
- **고용 상담** — 사람 상담 채널

### 외고반장은 매칭이 끝난 다음을 잡음

- 신규 인력 요청서 자동 작성 → 송출회사/행정사 전달
- 기존 직원 체류만료 D-day 모니터링
- 누락 서류 탐지
- 다국어 서류 요청 메시지 (승인 후 발송)
- 응답 해석 → 상태 업데이트
- 행정사 전달 패키지 + 감사 로그

### 겹치는 부분과 안 겹치는 부분

| 영역 | 워크비자 | 외고반 |
| --- | --- | --- |
| 인재풀 검색·매칭 | 코어 기능 (5,625명 DB) | 매칭 에이전트 일부만 |
| 신규 인력 요청서 생성 | 채용 공고 양식 (수동) | 자연어 → 자동 구조화 |
| 비자 진단 | 외국인 셀프 진단 | 사업장 케이스별 D-day 관리 |
| 다국어 메시지 작성 | 없음 | 코어 기능 |
| 체류만료/서류 누락 모니터링 | 없음 | 코어 기능 |
| 행정사 전달 패키지 | "상담"으로 분리 | 자동 패키징 |
| 감사 로그·승인 흐름 | 없음 | 코어 기능 |
| UX | 메뉴 클릭형 SaaS | 자연어 입력 Agentic OS |

## 3. 행정사와 외고반장의 차이

- **회계사** ↔ **삼쩜삼 / 자비스** — 회계사가 사라진 게 아니라, 평소 영수증 정리·간단한 신고는 앱이 하고, 복잡한 건 회계사가 함
- **변호사** ↔ **로톡 / 모두싸인** — 변호사가 사라진 게 아니라, 표준계약서·간단 상담은 플랫폼이 하고, 소송은 변호사가 함

### 행정사가 실제로 하는 일과 안 하는 일

- **정부 포털 직접 제출** (EPS, 하이코리아 등에 사장님 대신 신청)
- **법적 책임이 있는 자문** (이 케이스는 연장 가능합니다 같은 판정)
- **복잡한 케이스 처리** (사업장 변경, 분쟁, 이의 신청)
- **정부와 직접 소통** (고용센터 방문, 보완 요청 응대)
- **제출 후 결과 책임** (불허가 시 이의신청까지)

### 행정사가 안 하는 일 (=외고반장이 함)

행정사는 **케이스가 도착하면 처리하는 사람**이지, **케이스를 미리 챙겨주는 사람** 이 아님.

- 직원 50명 만료일 매일 모니터링 → ❌ 행정사 안 함 (사장님이 알려줘야 함)
- 사장님과 외국인 직원 사이 다국어 소통 → ❌ 행정사 안 함 (사장님이 알아서)
- 누락 서류 사전 탐지 → ❌ 행정사 안 함 (사장님이 다 모아서 가야 검토)
- 카톡으로 사장님에게 알림 → ❌ 행정사 안 함
- 이번 달 처리할 일 브리핑 → ❌ 행정사 안 함

```jsx
[사장님이 케이스 발생 인지]
         ↓
┌──────────────────────────────┐
│  외고반장: 운영 영역             │
│  • D-day 모니터링                │
│  • 다국어 소통                   │
│  • 서류 취합·누락 탐지            │
│  • 패키지 자동 생성               │
│  ※ AI는 판정하지 않음           │
└──────────────────────────────┘
         ↓ [완결된 패키지 전달]
┌──────────────────────────────┐
│  행정사: 전문 영역               │
│  • 법적 자문 / 케이스 판정        │
│  • 정부 포털 제출                 │
│  • 보완 요청 응대                 │
│  • 분쟁 / 이의신청                │
└──────────────────────────────┘
         ↓
[정부 시스템 (EPS, 하이코리아)]
```

### 사장님이 실제로 겪는 일

```jsx
[1월] 사장님이 직원 명단 보고 "어... Nguyen 비자 언제 끝나지?"
       → 엑셀 뒤지거나 직원에게 직접 물어봄
       
[2월] "Nguyen 비자가 6월 만료네. 행정사한테 연락해야지"
       → 행정사: "여권사본, 외국인등록증, 표준근로계약서, 
                  고용허가서 사본 보내주세요"
       
[3월] 사장님이 Nguyen에게 한국어로 카톡: 
       "여권 사본 보내줘"
       → Nguyen이 베트남어로 답장: "..."
       → 사장님 "뭐라는 거야?"
       → 옆에 있는 한국어 잘하는 베트남 직원에게 물어봄
       
[3월 말] 서류 다 모았다고 생각하고 행정사한테 보냄
       → 행정사: "표준근로계약서 작년 거예요. 갱신본 필요해요"
       → 다시 처음부터
       
[4월] 행정사가 정부 포털 제출
       → 5월 결과 통보
       
[6월] 다음 직원 Tran의 만료가 다가옴 → 1월처럼 다시 시작
```

- 여기서 외고반장이 들어가는 곳은 행정사 앞 단계.
    
    ```jsx
    [1월] 외고반장 알림: "이번 달 위험 4건 발견. 가장 시급: Nguyen D-90"
           
    [2월] 사장님: "Nguyen 비자 준비해줘"
          외고반장: "필요 서류 4종 자동 체크. 
                    표준근로계약서 작년 거라 갱신 필요.
                    Nguyen에게 베트남어로 요청 메시지 보낼까요?"
           
    [3월] 외고반장이 Nguyen 카톡으로 베트남어 메시지 발송 (사장님 승인)
          Nguyen 답장 → 외고반장이 한국어 요약
          "여권 보냄. 사진은 내일. 외국인등록증은 갱신 중."
           
    [3월 말] 외고반장: "서류 4종 취합 완료. 행정사 전달 패키지 PDF 생성"
           → 행정사한테 깔끔한 PDF + 메일로 자동 발송
           
    [4월] 행정사: "제대로 정리돼서 왔네. 바로 검토 가능"
           → 정부 포털 제출
    ```
    
    | 항목 | 행정사 | 외고반장 |
    | --- | --- | --- |
    | **역할** | 전문가 (제출·자문) | 운영 매니저 (준비·모니터링) |
    | **타이밍** | 사장님이 부탁할 때 | 상시 (24/7) |
    | **법적 책임** | 있음 (전문 자격) | 없음 (사람 승인 전제) |
    | **D-day 모니터링** | ❌ | ✅ |
    | **다국어 소통** | ❌✅ | ✅ |
    | **누락 서류 사전 탐지** | ❌ | ✅ |
    | **정부 포털 제출** | ✅ | ❌ (안 함) |
    | **비자 가능 여부 판정** | ✅ | ❌ (안 함) |
    | **카톡 진입** | ❌ | ✅ |
    | **비용** | 케이스당 38만 원 + | 월 구독 / 케이스당 |
