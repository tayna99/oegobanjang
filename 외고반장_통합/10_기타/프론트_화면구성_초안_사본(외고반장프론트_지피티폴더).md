# 프론트 화면 구성

# 1. 전체 화면 구조

## 1.1 기본 원칙

- 역할에 따라 다르게.
    
    관리자/인사팀/행정 담당자는 PC 대시보드 중심.
    
    사업자/대표자는 모바일 브리핑 챗 중심.
    
    행정사는 검토 자료 확인 중심.
    
- 모든 업무: 오늘 할 일 큐로 돌아오기
    
    예를 들면 체류기간 임박, 계약 종료 임박, 누락 서류, 승인 대기, 행정사 검토 준비, 메시지 응답 도착 같은 것들.
    
- 에이전트가 어떤 다음 액션을 준비했는지 보여줌
    
    각 업무를 누르면 AI가 어떤 액션을 준비했는지 보여줘야 해.
    
    예를 들면:
    
    “Nguyen V. 체류기간 D-30”
    → Visa Document Agent가 필요 서류 확인
    → Multilingual Contact Agent가 베트남어 요청 메시지 초안 생성
    → Workforce Agent가 케이스 상태 업데이트
    → 승인 요청 생성
    
- 판단 기록: 왜 이 판단을 했는지 기록이 남아야 함

## 1.2 화면 연결

1. PC 대시보드에서 오늘 할 일 확인
“체류기간 임박 4명, 서류 보완 필요 7건이 있습니다.”

2. Nguyen 케이스 클릭
오른쪽 상세 패널 열림.

3. AI가 준비한 액션 확인
“베트남어 서류 요청 메시지 초안 생성 완료.”

4. 관리자 버튼 클릭
“대표 승인 요청 보내기.”

5. 모바일 화면 전환
대표자에게 브리핑 도착.

6. 대표자가 초안 확인
베트남어 원문과 한국어 번역 확인.

7. 대표자 승인
“이 메시지로 컨택할까요?” 클릭.

8. 결과 저장
“업무 기록 #4789에 저장되었습니다.”

**관리자 PC 화면**

```jsx
1. 상단바
- 로고
- 사업장 선택
- 검색
- AI 반장에게 물어보기
- 알림
- 사용자

2. 탭 메뉴
- 오늘 할 일: 오늘 처리해야 하는 업무 큐. 데모의 메인 화면.
- 근로자: 외국인 근로자 목록과 체류/계약 상태.
- 케이스: 채용, 체류연장, 계약종료, 서류보완 같은 사건 단위 관리.
- 메시지 관리: 근로자에게 보낸 메시지, 응답, 다국어 초안 관리.
- 행정사 검토 : 행정사에게 넘길 검토 자료, 전달 패키지, 승인 상태.
- 판단 기록: AI가 어떤 근거로 판단했는지, 누가 승인했는지, 언제 발송했는지.

3. 오늘 처리할 업무 카드: 오늘 처리할 외국인 고용 업
- 체류기간 임박
- 서류 보완 필요
- 신규 채용 준비
- 컨택 대기
- 응답 도착
- 승인 대기
- 행정사 검토 준비

		예시:
		
		체류기간 임박 4명
		서류 보완 필요 7건
		신규 채용 준비 1건
		컨택 대기 4건
		응답 도착 2건
		승인 대기 5건
		행정사 검토 준비 2건

4. 업무 큐 테이블
- 업무
- 대상
- 사업장/라인
- 상태
- 기한
- 다음 처리

예시 행:

체류기간 연장 서류 요청 / Nguyen V. / 화성 2공장 조립라인 / 우선 확인 / D-30 / 높음 / 초안 보기 / 김대리
신규 베트남 E-9 3명 채용 요청 / 송출회사 요청서 / 화성 1공장 / 준비 중 / 이번 주 / 중간 / 요청서 보기 / 박대리
후보자 입국 전 서류 요청 / Candidate A / 도장라인 / 승인 대기 / 5/20 / 중간 / 승인 요청 / 김대리
계약 종료 확인 / Tran T. H. / 포장라인 / 응답 도착 / 5/12 / 낮음 / 응답 요약 / 김대리

버튼: 처리 하기, 초안 보기, 요청서 보기, 승인 요청, 응답 요약, 검토 자료 만들기 

5. 오른쪽 상세 패널 (새 페이지로 이동하면 데모 흐름이 끊김)
- 왜 확인 필요?
- AI가 준비한 일
- 누락 서류
- 다음 할 일
- 업무 기록
- 대표 승인 요청 버튼

	예시 
	
	제목: Nguyen V. 체류기간 연장 서류 요청
	상태: D-30 / 우선 확인 / 승인 전
	대상: Nguyen Van A
	국적: 베트남
	체류자격: E-9
	소속: 화성 2공장 조립라인
	체류만료일: 2026.06.20
	계약종료일: 2026.07.01
	
	그 아래에 4개 블록.

		왜 확인이 필요한가요?
		체류기간 만료까지 30일 남았고, 표준근로계약서 사본과 여권 사본이 아직 확인되지 않았습니다.
		
		AI가 준비한 일
		베트남어 서류 요청 메시지 초안 생성
		한국어 번역본 생성
		행정사 검토용 자료 패키지 준비
		이전 대화 기록 확인
		
		다음 할 일
		Nguyen에게 서류 요청 메시지 발송
		응답 없을 경우 2일 뒤 리마인드
		서류 수신 후 행정사 검토 자료 생성
		
		판단 근거 / 업무 기록
		체류만료일: 2026.06.20
		최근 서류 상태: 표준근로계약서 사본 누락
		이전 메시지: 3일 전 서류 요청 기록 있음
		승인 전 외부 발송 제한 적용
```

# 2. 관리자용 대시보드

- 포인트: 행을 클릭 했을때 다음 처리가 열리는 구조
- 필수 요소: 사업장 선택
- 예시
    
    ```jsx
    외고반장
    한별제조 · 경기 화성 · 자동차부품
    근로자, 케이스, 서류, 메시지 검색
    [AI 반장에게 물어보기]
    알림 12
    김대리 / 인사팀
    ```

    - 외국인 근로자 목록, 체류자격, 체류만료, 계약종료일 확인.
- **케이스**
    - 체류연장, 신규채용, 계약종료, 서류보완 같은 사건 단위 관리.
- 메시지 관
    - 근로자에게 보낸 메시지, 응답, 다국어 초안 관리.

    - 행정사에게 넘길 자료, 검토 패키지, 승인 상태 관리.
- 판단 기록
    - AI 판단 근거, 사람 승인, 발송 이력 확인.

# 3. 사업자용 모바일 클라이언트: 단순화가 포인

- 모바일 클라이언트의 목적: 오늘 뭘 승인하면 되는지 알려주고, 승인 버튼을 누르게 한느 것.
- 승인할 내용이 먼저 보여야 함.
- 주요 구성: 오늘 브리핑, 모바일 챗, 승인 요청 카드, 위험 알림, 메시지 초안 확인, 간단한 “승인/수정/보류” 버튼

```jsx
┌────────────────────┐
│ 외고반장 | 오늘 브리핑 │
│ 한별제조 · 승인 필요 3건 │
├────────────────────┤
│ AI 반장 메시지        │
│ 대표님, 오늘 확인이 필요한 업무가 3건 있습니다. │
├────────────────────┤
│ 승인 카드 1           │
│ Nguyen V. 체류기간 연장 서류 요청 │
│ D-30 / 누락 서류 2건 / 승인 필요 │
│ [초안 보기] [승인하기] │
├────────────────────┤
│ 승인 카드 2           │
│ Candidate A 입국 전 서류 패키지 │
│ [검토 자료 보기] [승인하기] │
├────────────────────┤
│ 승인 카드 3           │
│ Tran T.H. 계약 종료 확인 │
│ [응답 요약 보기]       │
└────────────────────┘
```

- 상세 화면 구성:
    
    ```jsx
    예시)
    
    Nguyen V. 서류 요청 메시지
    
    체류기간 만료 D-30
    누락 서류 2건
    승인 후 Zalo로 발송됩니다.
    
    [AI가 확인한 내용]
    
    - Nguyen Van A / 베트남 / E-9
    - 3일 전 서류 요청 이력 있음
    - 표준근로계약서 사본 누락
    - 여권 사본 누락
    
    [메시지 초안]
    
    VN
    Xin chào Nguyen,
    vui lòng gửi bản sao hợp đồng lao động tiêu chuẩn và hộ chiếu.
    Cảm ơn bạn.
    
    KR
    안녕하세요 Nguyen 씨,
    표준근로계약서 사본과 여권 사본을 보내주세요.
    감사합니다.
    
    [예상 응답]
    긍정 응답 시: 서류 수신 후 검토 자료에 반영
    추가 질문 시: 필요한 서류 형식 안내
    응답 지연 시: 2일 뒤 리마인드 제안
    
    [수정 요청] [보내기 승인]
    ```
    

# 4. 공통 컴포넌트 구성

두 화면에서 반복해서 쓰는 UI 컴포넌트 정하기

예) 정상/ 주의/ 긴급/ 승인 대기 

- D- day 카드 : D-30, D-14, D-7 등
- 근로자 카드
- 서류 상태 카드
- 승인 요청 카드
- 판단 근거 박스
- 다음 할 일 추천 박스
- 메시지 추천 박스

- 컴포넌트 구조
    
    PC Dashboard 쪽:
    
    `AppShell`
    
    전체 레이아웃. 상단바, 사이드/탭, 메인 콘텐츠.
    
    `WorkspaceSelector`
    
    사업장/공장 선택.
    
    `GlobalSearch`
    
    근로자, 케이스, 서류, 메시지 검색.
    
    `TaskSummaryCards`
    
    체류기간 임박, 서류 보완 필요, 승인 대기 등 카드.
    
    `WorkQueueTable`
    
    오늘의 업무 큐.
    
    `TaskDetailPanel`
    
    행 클릭 시 열리는 오른쪽 상세 패널.
    
    `BriefingCard`
    
    오늘 브리핑.
    
    `ApprovalList`
    
    승인 대기 목록.
    
    `EvidenceTimeline` 또는 `WorkLogTimeline`
    
    판단 근거/업무 기록.
    
    모바일 쪽:
    
    `MobileBriefingShell`
    
    모바일 전체 프레임.
    
    `ChatBriefingMessage`
    
    AI 반장 브리핑 메시지.
    
    `ApprovalTaskCard`
    
    승인 필요한 업무 카드.
    
    `LiveAgentProgress`
    
    AI 처리 단계.
    
    `DraftPreviewCard`
    
    VN/KR 메시지 초안.
    
    `ScenarioCards`
    
    긍정 응답, 추가 정보 요청, 응답 지연 시나리오.
    
    `ApprovalActionBar`
