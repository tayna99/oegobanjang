# Why & 수요

- **큰 수요는 있다.**
    
    다만 사장이 SaaS 월구독을 직접 결제할 수요는 아직 검증 안 됐고,
    
    **행정사·송출회사·중소기업중앙회/협회 채널을 통한 수요가 더 현실적**
    
    - 근거
        - 정부는 2026년 고용허가제 E-9 쿼터를 8만 명으로 확정했고, 그중 업종별 쿼터 7만 명 중 제조업이 5만 명으로 가장 큼. 이 쿼터는 산업별 인력수급 전망, 사업주·관계부처·지자체·사업주단체 대상 현장 수요조사를 종합해 정한 것. 즉, 제조업 외국인력 수요 자체는 정책적으로 확인된 시장
        - 2026년에는 총 5회 고용허가 신청·접수가 예정되어 있고, 1회차만 봐도 전체 15,784명 중 제조업이 11,275명이다. 비수도권 제조업체 추가 고용 한도도 기존 20%에서 30%로 상향됐다. 이건 제조업 현장에서 외국인력 운영이 계속 늘거나 최소 유지될 구조라는 뜻
        - 중소기업중앙회 공지 기준으로도 2026년 1회차 신규 외국인근로자 신청은 워크넷 내국인 구인등록 7일 → 신청서류 접수 → 합격자 발표 → 고용허가서 발급 → 수수료 납입 같은 절차로 진행된다. 신청수수료도 1인당 38만 원으로 안내되어 있다. 이 말은 이미 “사람 1명 뽑을 때마다 행정 절차와 비용이 붙는 시장
        - 공공데이터포털에는 고용노동부의 E-9 외국인근로자 근무 현황 데이터가 국적별·지역별 통계로 공개되어 있음. 즉 시장을 지역·국적 기준으로 쪼개서 타깃팅할 수 있는 데이터 기반도 있음.
- 수요 강도
    
    
    | 기능 | 수요 가능성 | 이유 |
    | --- | --- | --- |
    | 비자 만료 알림 | 높음 | 놓치면 리스크가 크고 이해하기 쉬움 |
    | 제출 서류 누락 점검 | 높음 | 행정사·담당자 모두 바로 체감 가능 |
    | 다국어 서류 요청 메시지 | 중상 | 현장 pain은 있지만 “돈 낼 기능”으로 포장 필요 |
    | 추천 인재 | 중간 이하 | 후보 데이터 없으면 약하고, 차별/편향 리스크 있음 |
    | 인재 컨택 | 중간 | E-9 구조상 자유 채용/스카우트처럼 만들기 어려움 |
    |  |  |  |
- 결: 수요는 **외국인 채용 추천 AI에 있는 게 아니라, 외국인 고용 운영 사고를 줄이는 AI에 있음.**

- 쓸 이유
    1. 돈을 아껴준다
    2. 사고를 막아준다
    3. 귀찮은 연락을 줄여준다
    4. 행정사에게 넘기기 쉽게 정리해준다
    5. 외국인 직원과 말이 통하게 해준다
    6. 돈을 아껴준다
    7. 사고를 막아준다
    8. 귀찮은 연락을 줄여준다
    9. 행정사에게 넘기기 쉽게 정리해준다
    10. 외국인 직원과 말이 통하게 해준다

- 시나리오를 써보면
    
    ```jsx
    사장:
    “이번 달 외국인 직원 중에 처리할 거 있어?”
    
    AI:
    이번 달 처리 필요 업무 4건입니다.
    
    1. Nguyen Van A — 체류만료 D-76
       - 표준근로계약서 없음
       - 고용허가서 사본 없음
       - 베트남어 서류 요청 메시지 생성 가능
    
    2. Tran Thi B — 계약종료일 D-45
       - 체류만료일과 계약종료일 충돌 가능
       - 행정사 검토 필요
    
    3. 신규 베트남 근로자 3명 채용 요청
       - 송출회사 전달용 요청서 생성 완료
       - 후보자 서류 준비도 확인 필요
    
    승인할 작업:
    [서류 요청 메시지 발송]
    [행정사 패키지 생성]
    ```
    
- 이렇게 됐을때 사장이 해야할 일 명확해짐
    
    ```jsx
    - 뭘 놓쳤는지 알 수 있음
    - 누구에게 연락해야 하는지 알 수 있음
    - 외국어 메시지를 직접 안 써도 됨
    - 행정사에게 넘길 자료가 정리됨
    - AI가 최종 판단하지 않고 승인만 받음
    ```
    

- 기능별로 사용 가능성 시나리오
    1. 비자/서류 운영 에이전트 - 사용 가능성 높음
    
    ```jsx
    - 체류만료 놓치면 리스크 큼
    - 서류 누락은 자주 발생
    - 행정사에게 넘기기 전 정리가 귀찮음
    - 직원 수가 많아질수록 엑셀 관리가 불안정해짐
    ```
    
    - 사용자가 “이번 달 위험한 직원 찾아줘”라고 하면, AI가 D-day 계산, 누락 서류 확인, 다국어 요청 메시지, 행정사 패키지까지 경로를 짜주기.

1. 다국어 컨택 에이전트
- 외국인 근로자 관리에서 의사소통 문제는 실제 애로로 계속 언급됨. 동아일보가 중소기업중앙회 2026년 1월 조사 결과를 인용한 보도에 따르면, 외국인 근로자 관리 시 가장 큰 어려움으로 의사소통을 꼽은 응답이 52.1%였고, 의사소통 애로 중에는 작업 지시 오해로 인한 생산 차질이 63.9%로 가장 높음.
    
    ```jsx
    나쁜 기능:
    한국어를 베트남어로 번역해줌
    
    좋은 기능:
    서류 요청 목적, 제출 기한, 개인정보 사용 목적, 담당자 확인 포인트를 포함해
    업무용 메시지로 만들어주고,
    답변이 오면 서류 상태를 업데이트해줌
    ```
    

1. 인력 확보 에이전트 -사용 가능성 중간
- 여기서 말하는 진짜 추천이라면 이런 데이터가 필요.
    
    ```jsx
    - 실제 후보자 풀
    - 송출회사 데이터
    - 입국 가능 시점
    - 서류 준비 상태
    - 직무 경험
    - 언어 수준
    - 근무조건 이해 여부
    - 고용허가 가능 여부
    ```
    

- 고용주가 쓰게 하려면 화면보다 상황 트리거가 중요함
    
    고용주는 매일 들어와서 AI에게 말하지 않을 수 있음. 진짜 쓰게 하려면 AI가 먼저 이렇게 알려줘야 해.
    
    ```
    이번 주 처리 필요 업무 3건입니다.
    
    1. Nguyen — 체류만료 D-76
    2. Tran — 계약종료 D-45
    3. 신규 후보자 2명 — 서류 미제출
    
    처리할까요?
    ```
    
    즉 UX는 
    
    ```
    1. 시스템이 먼저 위험을 감지
    2. 카톡/문자로 알려줌
    3. 고용주는 “처리해줘”라고 답함
    4. AI가 실행 계획을 보여줌
    5. 고용주가 승인함
    6. 메시지/패키지/로그가 생성됨
    ```
