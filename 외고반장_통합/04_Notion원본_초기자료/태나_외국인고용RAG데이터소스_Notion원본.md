# [태나] 외국인 고용 RAG 데이터 소스

관련 사용자: 태나
단계: MVP
마감: 2026년 4월 29일
상태: 아이디어
시작: 2026년 4월 29일
우선순위: P0
유형: 리서치

법령·절차·서식·안전자료는 RAG에 넣고, 직원 상태·후보 상태·서류 보유 여부·D-day 계산은 DB/룰베이스로 빼야함.

- RAG = 공식 근거와 절차를 찾는 곳
- SQL = 현재 직원/후보자 상태를 저장하는 곳
- Rule = 날짜 계산과 true/false 판단을 하는 곳
- LLM = 자연어 구조화, 요약, 메시지 생성, 설명을 하는 곳

## 1. RAG 설계

- 첫째, **공식 근거 찾기**.
    
    예를 들어 “E-9 고용 절차가 어떻게 되지?”, “고용허가서 발급 전에 뭘 준비해야 하지?”, “고용변동 신고는 어떤 경우에 해야 하지?” 같은 질문에 공식 문서 기반으로 답해야 함. EPS/HRD Korea는 사업주 고용절차에서 내국인 구인노력, 고용허가서 발급, 표준근로계약 체결, 사증발급인정서 신청 등 흐름을 설명
    
- 둘째, **서류 체크리스트 만들기**.
    
    예를 들어 “체류기간 연장”, “고용변동 신고”, “신규 E-9 고용허가 신청” 같은 케이스별로 필요한 서류와 확인 항목을 뽑아야. 정부24의 체류기간연장허가 안내에는 여권, 외국인등록증, 체류지 입증 서류 등 제출 항목이 안내되어 있고, 법령정보센터 서식에는 외국인근로자 고용변동 등 신고서와 유의사항이 포함되어 있음.
    
- 셋째, **업무 메시지 생성 근거 제공**.
    
    다국어 컨택 에이전트가 “서류를 보내주세요”라고만 번역하면 부족. 왜 이 서류가 필요한지, 언제까지 필요한지, 개인정보는 어디에 쓰는지, 담당자 승인 후 발송해야 하는지까지 메시지에 반영해야 함.
    
- 넷째, **행정사 전달 패키지의 근거 붙이기**.
    
    행정사에게 넘길 때 “AI가 이렇게 말했습니다”가 아니라, “이 직원은 D-76이고, 표준근로계약서와 고용허가서 사본이 누락되어 있으며, 관련 검토 항목은 이것입니다” 식으로 근거와 상태를 묶어야 함.
    

# 전체 구조

```jsx
[User Goal]
“베트남 E-9 근로자 3명 추가 채용 준비해줘.
Nguyen 체류만료도 확인해줘.”

↓ Intent Router

[1] 인력 확보 Agent
→ RAG: E-9 고용절차, 허용업종, 신청 흐름
→ SQL/Mock: 회사 정보, 후보자 정보

[2] 다국어 컨택 Agent
→ RAG: 메시지 템플릿, 개인정보 안내 문구, 안전/생활 안내 자료
→ LLM: 한국어 원문 + 베트남어 번역
→ Rule: 승인 전 발송 금지

[3] 비자·서류 운영 Agent
→ SQL: 직원 상태, 체류만료일, 계약종료일, 보유 서류
→ Rule: D-day 계산, 누락 서류 비교
→ RAG: 관련 법령, 서식, 절차, 제출 서류
→ Output: 행정사 패키지

↓ Human Approval

↓ Evidence Log
```

## 2. RAG 인덱스 & 공식 데이터 소

### 1) 규정 RAG

여기에는 법령, 고용허가제 절차, 체류 절차, 서식 안내

사용하는 에이전트:

```jsx
- 인재 요건·매칭 에이전트
- 후보 적합성 검토 에이전트
- 비자·체류 관리 에이전트
- 서류 체크리스트·패키징 에이전트
- 사람 승인·전문가 전달 에이전트
```

들어갈 데이터:

```jsx
- 출입국관리법
- 외국인근로자의 고용 등에 관한 법률
- 고용허가제 EPS 안내
- 사업주 고용절차
- E-9 허용업종
- 고용변동 신고서
- 체류기간 연장 관련 안내
- 고용허가 신청 관련 안내
```

| 소스 | 용도 |
| --- | --- |
| 국가법령정보센터 law.go.kr | 출입국관리법, 외국인근로자고용법, 시행령, 시행규칙, 서식 |
| EPS / 고용24 | 고용허가제 절차, 허용업종, 사업주 신청 흐름 |
| 한국산업인력공단 EPS 안내 | 사업주 고용절차, 근로계약, 사증발급인정서 흐름 |
| 정부24 | 체류기간연장허가 등 민원 안내 |
| 고용노동부 보도자료 | 연도별 신청 일정, 쿼터, 제도 변경 |
|  |  |

### 2) 서류 및 케이스 템플 RAG

- 서식, 체크리스트, 샘플 케이스, 행정사 전달 템플릿
- 사용하는 에이전트
    
    ```jsx
    - 제출 서류 간소화 에이전트
    - 비자·체류 관리 에이전트
    - 사람 승인·전문가 전달 에이전트
    - 감사 로그·근거 저장 에이전트
    ```
    
- 들어갈 데이터:
    
    ```jsx
    - 고용변동 등 신고서 PDF
    - 고용사업장 정보변동 신고서
    - 체류기간 연장 관련 제출 서류
    - 고용허가 신청 관련 준비 서류
    - 표준근로계약서 관련 안내
    - 행정사 전달 패키지 템플릿
    - synthetic 케이스 20~50개
    ```
    
- 데모용 합성 데이터/ 합성 케이스
    - 답변 근거 x 그냥 데모용 케이스.
    
    ```jsx
    합성 케이스:
    E-9 베트남 근로자 Nguyen Van A
    체류만료 D-76
    여권 있음 / 외국인등록증 있음 / 표준근로계약서 없음 / 고용허가서 사본 없음
    
    정답:
    - 누락 서류: 표준근로계약서, 고용허가서 사본
    - 다음 행동: 베트남어 서류 요청 메시지 생성
    - high-risk 여부: 행정사 검토 필요
    ```
    
    - 고용변동 신고서 양식에는 국적, 여권번호, 외국인등록번호, 체류자격, 체류기간, 사유 발생일, 발생 사유 같은 필드가 들어가고, 변경 사실을 안 날부터 15일 이내 신고하라는 유의사항도 포함
    - 서류 패키징과 필드 누락 탐지에 바로 쓸 수 있음.

### 3) 다국어 컨택 RAG

- 단순 번역기와 차별화하려면 “업무용 메시지 템플릿”
    
    사용하는 에이전트:
    
    ```jsx
    - 다국어 컨택·온보딩 에이전트
    - 응답 해석·다음 행동 에이전트
    - 비자·서류 운영 에이전트
    ```
    
    들어갈 데이터:
    
    ```jsx
    - 서류 요청 메시지 템플릿
    - 입국 전 안내 메시지 템플릿
    - 기숙사 안내 메시지 템플릿
    - 근무조건 확인 메시지 템플릿
    - 개인정보 사용 목적 안내 문구
    - 안전교육 안내 문구
    - 급여명세서 설명 템플릿
    - 다국어 안전표지/안전교육 자료
    ```
    
    외부 데이터 소스 
    
    | 소스 | 용도 |
    | --- | --- |
    | KOSHA / 안전보건공단 | 외국인 노동자 안전교육, 다국어 안전자료 |
    | 중소기업중앙회 외국인력지원실 | 다국어 안전표지 자료 |
    | 외국인력상담센터 | 상담 언어, 지원 채널 안내 |
    | 외국인노동자지원센터 자료 | 상담/생활 이슈 참고 |
    | 자체 작성 템플릿 | 서류 요청, 기숙사 안내, 근무조건 확인 메시지 |

## 4. RAG말고 SQL/DB로 관리할 데이터

```jsx
- 직원명
- 외국인등록번호
- 여권번호
- 체류만료일
- 계약종료일
- 입국일
- 보유 서류 true/false
- 후보자 개인정보
- 담당자 연락처
- 발송 기록
- 승인 기록
```

```jsx
Postgres / SQLite
- companies
- employees
- candidates
- documents
- cases
- approvals
- audit_logs

Vector DB / RAG
- laws
- procedures

```

## 검색기

```jsx
Retriever 1: RegulationRetriever
법령/절차/서식 검색

Retriever 2: TemplateRetriever
메시지/안전/생활 안내 검색

Retriever 3: CaseRetriever
합성 케이스/내부 케이스 검색
```

AI가 위험한 판단을 하지 않도록, 모든 제안 뒤에 공식 근거와 사람 승인 지점을 붙여주는 안전장치.

```jsx
공식 근거:
법령, EPS, 정부24, HiKorea, 고용노동부, KOSHA

업무 템플릿:
서류 요청, 온보딩, 안전교육, 행정사 패키지

상태 데이터:
직원 CSV, 후보자 CSV, 보유 서류, D-day

평가 데이터:
합성 케이스, 누락 서류 테스트, 메시지 테스트
```
