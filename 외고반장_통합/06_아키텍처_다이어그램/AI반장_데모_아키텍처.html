Briefing 카드 2-Step 흐름

Step A · 0~10초
📋 Briefing
브리핑 생성 버튼 → 라우팅/실행 → 브리핑 카드 1장

요약(1문장) · Top 3 · 리스크 · Next Action 버튼들
Step B
⚡ Execute
Next Action 클릭 → Plan 생성 → (승인) → 실행 → Evidence/Log 기록

📨 표준 이벤트(Envelope): 버튼이 만드는 입력 규격

브리핑 버튼은 "명령어"가 아니라 컨텍스트 스냅샷 이벤트를 만든다.

필드예시값설명
trigger_type briefing_button 입력 텍스트 없어도 OK
persona 근로자 행정사 인력담당자 라우팅 1순위 feature
policy Daily Risk Submission Outreach 버튼 옵션 (기본값: Daily)
case_url 현재 보고 있는 케이스 항상 포함 (핵심 컨텍스트)
org_url 회사/고객 URL 템플릿/권한 스코프
language ko / vi / en 다국어 에이전트 입력
constraints needs_approval=true 외부 발송/제출은 승인 필수

라우팅 우선순위

  • 1
    Rule layer (결정적)

    금지/필수/우선순위/승인 조건 — 항상 최우선 적용

  • 2
    LLM layer (보조)

    상황 요약, Top 3 선택, 문구 구성 — Rule 이후 보완

  • 3
    Fallback

    Rule만으로도 최소 브리핑 가능하도록 보장
