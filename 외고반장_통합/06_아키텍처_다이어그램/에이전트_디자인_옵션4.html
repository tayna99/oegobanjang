중간 — 모든 에이전트의 공통 내부 구조

executor_node에서 호출되는 에이전트는 동일한 3축 구조 = 서브에이전트 + Tool 4단계 + Middleware 7개

① 서브에이전트 (각 2개씩)

각자 독립 LLM ReAct 루프
최대 4회 반복 (ToolCall 12회 한도 별도)
결과 통합 후 risk_flags dedupe
evidence_events 누적
에이전트별 매핑 예시
· Visa: visa_risk + document_priority
· Workforce: requirement + readiness
· Contact: message_draft + reply_summary

② Tool 4단계 (총 20개)

READ DB / RAG 조회만 (8개)
CALC 날짜 · 우선순위 · 검증 (6개)
DRAFT 초안 · 항상 approval=True (3개)
APPROVAL 즉시 NEEDS_APPROVAL 반환 (3개)
중앙 레지스트리 관리
· tools/registry.py
· TOOL_REGISTRY[name] → (tool, grade)
· grade 기반 자동 라우팅

③ Middleware 7개

1. WorkBridgeSafety — 의도 차단
2. EvidenceCapture — 이벤트 수집
3. PIIMiddleware(email)
4. PIIMiddleware(foreign_hiring_pii)
5. ModelCallLimit(12)
6. ToolCallLimit(12)
7. HumanInTheLoop — 강제 중단
전 에이전트 동일 스택 · 순서도 동일
에이전트 = (서브에이전트 2개 × Tool 4단계 × Middleware 7개) 조합으로 정의
