## 핵심 아이디어

외국인을 고용한 사업장이 직원의 체류자격, 비자 갱신, 서류 준비, 기관 예약, 다국어 안내를 한 번에 관리하는 서비스. 쉽게 말하면: **외국인 직원 HR/체류관리 전용 AI 비서**

이 아이디어는 이후 팀 논의를 거쳐 "외고반장(외국인 고용 운영 Agentic OS)"으로 발전했다.

## 주요 사용자
- 외국인 고용 사업장 / 인사팀·총무팀 / 외국인 근로자 / 외국인 인력 매칭 기업 / 행정사·노무사 사무소

## 기본 플로우
```
직원 정보 입력
→ 체류자격 판단
→ 필요한 서류 체크리스트 생성
→ 비자 갱신 일정 계산
→ 기관별 예약 안내
→ 다국어 번역/알림
→ 진행 상태 관리
```

## 핵심 기능(초기 구상)

### 1. 체류자격 판단
입력 예시: 국적(베트남) / 직무(생산직) / 업종(제조업) / 현재 비자(E-9) / 체류 만료일(2026-08-20) / 근무지(경기도)
출력 예시: "현재 E-9 체류자격 유지 가능성이 높습니다. 만료일 기준 4개월 전부터 갱신 준비가 필요합니다."

### 2. 비자 갱신 스케줄 자동 생성
```
D-120: 갱신 준비 시작
D-90: 서류 수집
D-60: 기관 예약
D-30: 최종 확인
D-7: 미제출 서류 알림
```

### 3. 서류 체크리스트 자동 생성
비자 종류·업종·국적·근무지에 따라 조건 분기. 예: E-7 전문인력 — 여권 사본, 외국인등록증, 고용계약서, 사업자등록증, 학위증명서, 경력증명서, 납세증명서

### 4. 기관별 예약 안내
하이코리아, 출입국·외국인청, 고용센터 등. 직접 예약 자동화는 위험 → "예약 가이드 + 필요 링크 + 준비물 안내" 수준으로 제한 (이 원칙은 이후 "정부 포털 자동 제출 금지" 원칙으로 발전).

### 5. 다국어 안내
한국어 → 영어/베트남어/중국어/태국어/인도네시아어

## 추가 아이디어 (초기 구상, 일부는 이후 채택/일부는 폐기)

1. **직원별 체류 리스크 점수**: 만료일/서류누락/예약미완료 기준 위험도 표시(안전/주의/위험) — 이후 "위험도 판단은 AI가 확정하지 않고 룰 기반으로 표면화" 원칙으로 순화되어 채택됨
2. **서류 누락 자동 검출**: 업로드 PDF/이미지 분석 → 이후 Document Gap Case로 발전
3. **서류 유효기간 검사**: 발급일 기준 재발급 필요 여부
4. **직원용 모바일 안내 페이지**: 제출 서류/만료일/다음 할 일/담당자 메시지 → 이후 모바일 브리핑 화면으로 발전
5. **담당자 업무 대시보드**: 전체 외국인 직원 현황 요약 → 이후 오늘 브리핑/PC 대시보드로 발전
6. **LangGraph 조건 분기 데모**: 비자 종류 → 업종 → 국적 → 거주지 → 필요 서류 결정 → 일정 생성 → 알림 발송
7. **법령 RAG 검색**: 출입국관리법, 법무부 체류자격 안내, 고용허가제 안내 문서 벡터DB화
8. **변경 고시 감지**: 법무부/하이코리아 공지사항 주기적 확인 → 이후 Issue Log의 "법령 버전 관리" 이슈로 이어짐
9. **회사별 템플릿 관리**: 반복 안내문/서류 요청 문구 저장
10. **상담 기록 Memory**: 직원별 대화 이력 장기 저장(LangGraph Checkpoint/Memory)

## LangGraph 패턴 구상 (태나 메모, 2026-04-24)

- StateGraph + 장기 상태 관리 (Long-lived State)
- Checkpoint의 본격 활용 — "HITL 승인 대기 중 상태 보존" 패턴 예시:
  - 베트남 노동자가 E-9 비자로 한국 도착 → 3개월 후 건강검진 알림 → 11개월 후 비자 갱신 서류 준비 안내 → 2년 10개월 후 재고용 여부 상담 → 4년 10개월 후 출국·재입국 절차
- Conditional Edge의 대량 분기 (비자 종류 E-7/E-9/F-2 등 × 업종 × 국적 × 거주지 = 총 분기 경로 20~30개)
- Supervisor + Subagent 패턴(초기 구상):
  ```
  [Supervisor Agent]
  ├─ [Visa Classifier Agent]      체류자격 분류·검증
  ├─ [Document Checklist Agent]   서류 목록 생성·상태 추적
  ├─ [Deadline Tracker Agent]     기한 계산·알림
  ├─ [Appointment Booker Agent]   하이코리아·은행·병원 예약
  ├─ [Translator Agent]           5개 언어 번역
  ├─ [Life Guide Agent]           주거·의료·세금·교육 안내
  └─ [Compliance Checker Agent]   법령·내규 준수 검증
  ```
  → 이후 실제 구현에서는 3 Mission Agent(인력확보/다국어컨택/비자서류) + 8 Sub-agent 구조로 정리됨 (`2026-07-04_v1_세에이전트_수렴아키텍처.md` 참고)
- Tool Use(Function Calling) — 외부 API 통합
- 인터럽트 + 휴먼 개입: 비자 갱신 서류 제출 최종 단계(담당자 최종 확인 필수), 노동계약서 검토 후 서명(법무·노무사 승인), 가족 초청 F-3 신청(당사자 최종 확인) → 이후 Human Approval Gate 원칙으로 일반화됨
- Scheduled/Cron Trigger: 비자 만료 6개월 전 자동 알림, 건강검진 기한 2주 전 자동 예약 유도, 연말정산 시즌 자동 안내 → 이후 Daily Briefing Scheduler로 발전
