## 에이전트별 Rule-base로 계산/판정할 것들

### 1. 인력확보(Workforce) 에이전트

**케이스 생성 조건**
- 채용 필요 발생 트리거(라인 증설, 결원, 이직률 임계치 등)
- "언제부터 급한가"(필요 시점 - 리드타임) → 우선순위 산정

**적합성/제약 체크(하드 룰)**
- 공장/사업장/테넌트별 정원, 직무 제한, 지역/기숙사 수용, 예산 상한
- E-9/비자/쿼터 등 "불가/제한" 조건(확정 판정은 룰로)

**중복 방지 / idempotency**
- 같은 결원·같은 포지션·같은 기간에 케이스 중복 생성 방지 키

**라우팅**
- 어떤 채널(에이전시/지인/플랫폼/내부풀)로 갈지 규칙 기반 1차 분기

**승인 게이트 조건**
- 외부 공고/컨택/에이전시 요청은 "승인 필요"로 강제
- LLM은 그 다음에: "포지션 요약", "요구사항 정리", "에이전시에게 보낼 브리프", "후보자/리드 요약" 같은 문장/문서 초안만 만들게 두는 게 안전

### 2. 비자 서류 에이전트

**데드라인 기반 임계값 판정**
- 체류만료 D-7 이내 → CRITICAL
- 체류만료 D-30 이내 → HIGH
- (임계값에 따라 우선순위/알림 강도를 정하는 룰)

**누락/충족 여부 체크(필수조건 룰)**
- 필수 서류 없음 → missing_document (예: "여권 사본/표준근로계약서 누락")

**데이터 간 충돌/정합성 룰**
- 계약종료일 < 체류만료일 → contract_visa_conflict 후보

### 3. 다국어 컨택(Multilingual Contact) 에이전트

**언어 선택/템플릿 선택**
- 근로자 선호 언어/국적/과거 대화 언어 → 언어 확정
- 케이스 타입(서류 요청/일정 안내/리마인드/경고) → 메시지 템플릿 분기

**민감도/리스크 분류**
- 법률 판단/비자 가능 "확정" 표현 금지 같은 금칙 룰
- 개인정보 포함 여부 감지(주민번호/여권번호 등) → 마스킹/차단

**발송 채널/시간 룰**
- WhatsApp/SMS/카카오/전화 등 채널 허용 정책
- 야간 발송 제한, 휴일 발송 제한, 리마인드 쿨다운(예: 24시간 내 재발송 금지)

**승인 게이트 조건**
- 특정 톤(경고/불이익 고지), 특정 케이스(체류/계약)에서는 무조건 승인 필요

**대화 상태 전이**
- "발송 예정 → 승인 대기 → 발송 완료 → 회신 대기 → 미응답 처리" 같은 상태 전이 규칙

## UI 정리 메모 (원본 "UI 정리" 페이지, 2026-05-15 작성)

> 아래는 초기 UI 작업 시점의 실무 todo 메모입니다. 이후 코드베이스 기반의 `02_UIUX설계/2026-05-13_UIUX_기획정리.md`, `2026-05-13_UIUX_코드기준_스크린샷정리.md`가 더 최신/상세하므로, 여기서는 당시 결정된 세부 UI 규칙만 보존합니다.

오늘 할 일 카드 관련:
- 상태·기한·업무만 나오게 (그 외 정보는 상세 패널로)
- [초안 보기] 클릭 → 컨택 메뉴 이동
- [요청서 보기] 클릭 → 채용 준비 메뉴 이동
- [승인 요청] 클릭 → 채용 준비 메뉴 이동
- [응답 요약] 클릭 → 컨택 메뉴 이동
- 다음 처리 열 정리 → 채용 준비

신규 E-9 채용 준비:
- 체크리스트 8개. [요청서 보기] 클릭 시 행정사 검토 메뉴 UI 노출. [행정사 검토 요청] 클릭 시 행정사 검토 메뉴의 이메일 보내기 기능 연결

입국 전 서류 패키지:
- 체크리스트 5개. 근로자 근속 열 제거

위험도 3단계 관리:
- **서류 누락 → 확인 필요**: 관리자 확인이 안 된 상태
- **정상**: 행정사까지 다 보낸 것, 검토 완료
- **확인 중**: 관리자가 확인하면 확인 중 상태로 전환
