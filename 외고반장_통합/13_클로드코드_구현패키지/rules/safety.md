# rules/safety — 도메인 안전 체크리스트 (승인·발송·PII 관련 작업 시 로드)

작업이 다음을 건드리면 이 체크리스트를 통과해야 커밋한다.

## 발송·승인 경로를 만들거나 수정할 때

- [ ] 외부로 나가는 함수가 새로 생겼는가? → 생겼다면 잘못 — `requestApproval()`로 회귀
- [ ] 승인 없이 상태가 human_approved가 되는 경로가 있는가? → 차단
- [ ] idempotency key 없이 decide()를 호출하는 곳이 있는가?
- [ ] 스트리밍 완료 전 승인 버튼이 활성화되는가?
- [ ] 오프라인에서 승인 버튼이 눌리는가?
- [ ] high risk 케이스에 handoff 외의 처리 버튼이 생겼는가?

## PII를 다룰 때

- [ ] fixture·스토어·로그·테스트에 등록번호/여권번호 원문이 있는가?
- [ ] 근로자 원문 메시지가 목록 미리보기·evidence 요약에 노출되는가?
- [ ] evidence에는 해시만 — payload 원문 저장 금지

## 문구를 추가할 때

- [ ] "가능합니다/불가능합니다" 단정 판정 표현이 있는가?
- [ ] "발송 완료"라고 썼는가? → "발송 승인 완료"
- [ ] 성실도·이탈·평가 뉘앙스 단어가 있는가?
- [ ] 국적이 색·순위·추천에 쓰였는가?

## Evidence를 다룰 때

- [ ] 상태 변경 액션인데 append를 빼먹었는가?
- [ ] 기존 이벤트를 수정·삭제하는 코드가 있는가?
- [ ] 사람 이벤트에 결정자·본인/대리 표기가 있는가?
