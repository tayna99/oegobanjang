## 1. 총평

기존 코드는 **PC 데모용 정적 프로토타입**이다. 화면 7종의 시각 골격과 mock 데이터는 쓸만하지만, 다음 4가지가 구조적으로 없다.

1. **라우터 없음** — `useState` 탭 전환뿐. URL이 없어 2단계 딥링크 맵(oegobanjang://case/{id})을 얹을 수 없다 → react-router 도입이 선행 조건
2. **모바일 없음** — `grid-cols-7`, 고정 320px 패널, 테이블 중심. breakpoint 분기 전무
3. **상태 없음** — loading / empty / error / offline 0개. 모든 데이터가 하드코딩 문자열 (D-day도 `'D-30'` 문자열)
4. **API·상태관리 레이어 없음** — data.ts 직접 import. Approval 상태 전이 불가

결론: **컴포넌트는 살리고 앱 골격은 새로 짠다.** "수정 후 재사용" 대부분은 토큰 교체 수준이라 비용이 낮다.

## 4. 마이그레이션 순서 (권장 4스텝)

```
Step 1  토큰 치환 (1~2일 규모)
        tailwind.config에 4단계 토큰 주입 → ui.tsx 4개 컴포넌트 수정
        → 기존 PC 화면 7종이 자동으로 새 룩으로 전환됨
Step 2  골격 교체
        react-router 도입, breakpoint 분기(모바일 쉘/PC 쉘), data.ts 스키마 재정의
Step 3  모바일 P0 화면
        M1 → BottomSheet/M2 → M3(ContactView 상세부 이식) → M4/M9 → M5
        (5단계 HTML 프로토타입의 마크업·모션 값을 그대로 React화)
Step 4  상태·온보딩
        상태 5종 인프라 → M6·M7(CasesView 하향)·M8(JudgmentLog 상세 하향) → 온보딩
```
