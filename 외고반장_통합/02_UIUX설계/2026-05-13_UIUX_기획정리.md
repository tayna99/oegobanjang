# 외고반장 UI/UX 기획 정리

> 작성일: 2026-05-13  
> 기준: 코드베이스 전체 분석 (frontend/, frontend-proto/, docs/, backend/ 포함)

---

## 1. 제품 한 줄 정의

E-9 외국인 근로자를 고용하는 제조업 중소기업이 **체류·고용·소통 리스크**를 놓치지 않도록 돕는 **AI 기반 외국인 고용 운영 OS**.

- 비자 신청 대행 ❌ → **운영 리스크 관리** ✅
- AI 단독 결정 ❌ → **초안 생성 + 담당자 승인** ✅
- 정부 포털 자동 제출 ❌ → **패키지 초안 + 행정사 전달** ✅

---

## 2. 사용자 분석

### 2-1. Primary 사용자

| 구분 | 역할 | 사용 맥락 | 주요 디바이스 |
|------|------|-----------|--------------|
| 인사 담당자 | 근로자 체류/서류/소통 전담 | 사무실 PC, 간헐적 모바일 | PC 우선 |
| 사장·현장 관리자 | 최종 승인 권한자 | 현장 이동 중 모바일 많음 | 모바일 우선 |

### 2-2. Secondary 사용자

| 구분 | 역할 | 인터페이스 접점 |
|------|------|----------------|
| 행정사·노무사 | 패키지 수신, 검토 | 행정사 전달 패키지 (PDF/문서 export) |
| 외국인 근로자 본인 | 서류 제출, 메시지 수신 | 모바일 채널 (Zalo, 카카오톡, 문자) - 앱 직접 사용 ❌ |

### 2-3. 사용자 환경 제약

- 한국어 중심 인터페이스 (담당자 대상)
- 근로자와의 소통은 베트남어(`vi`) 우선, 인도네시아어(`id`) 지원 예정
- 현장 관리자는 PC 접근 없이 모바일로만 **승인** 처리해야 함
- 인터넷 연결이 불안정할 수 있음 → 데모 데이터 fallback 배지 필요

## 3. 정보 구조 (IA)

```
외고반장
├── 오늘 할 일 (Dashboard)          ← 진입점, 매일 브리핑
├── 채용 준비 (Hiring)              ← 신규 E-9 채용 요청 관리
├── 근로자 (Workers)                ← 재직 근로자 목록/상세

```

## 4. 화면별 기획 요구사항

### 4-1. 오늘 할 일 (Dashboard) — 핵심 화면

**목적:** 출근 직후 오늘 처리할 업무를 한눈에 파악하고 우선순위를 정한다.

**상단 브리핑 카드 (요약 배너)**
- "외고반장이 N개 케이스를 정리했습니다" 문구
- 즉시 확인 / 우선 확인 / 승인 대기 건수 표시
- 생성 시각 표시
- `다시 생성` 버튼 (AI 재실행)

**요약 통계 카드 (7종, 가로 그리드)**

| 카드 | 아이콘 색상 | 단위 |
|------|------------|------|
| 체류기간 임박 | 빨강 `#EF4444` | 명 |
| 서류 보완 필요 | 주황 `#F97316` | 건 |
| 신규 채용 준비 | 초록 `#10B981` | 건 |
| 컨택 대기 | 보라 `#8B5CF6` | 건 |
| 응답 도착 | 하늘 `#0EA5E9` | 건 |
| 승인 대기 | 노랑 `#F59E0B` | 건 |
| 행정사 검토 준비 | 인디고 `#6366F1` | 건 |

**필터 버튼 행**
- 우선순위: 전체 / 즉시 확인 / 우선 확인 / 확인 필요 / 참고
- 상태: 전체 라인 / 승인 대기만 / 서류 보완 필요

**업무 큐 테이블 (메인)**

| 컬럼 | 설명 |
|------|------|
| 근로자 | 표시명 + 아바타 (PII 마스킹) |
| 국적·체류 | 국기 이모지 + 체류자격 |
| 체류만료 / D-day | 날짜 + D-N 뱃지 (색상 위험도 반영) |
| 계약 종료 | 날짜 |
| 서류 | 제출/미제출 비율 |
| 위험도 / 케이스 | CRITICAL/HIGH/MEDIUM/LOW 배지 |
| 다음 처리 | 추천 액션 버튼 |

**우측 슬라이드 패널 (상세)**
- 선택된 근로자/케이스 상세
- 추천 액션 목록
- 근거 자료 (RAG citation)
- 승인 대기 중인 액션 상태

**AI 채팅 패널 (우측 오버레이)**
- `?ai=1` 파라미터 또는 버튼으로 열기
- 자연어 질문 → Agent 응답
- 근거 출처 인라인 표시

---

### 4-2. 채용 준비 (Hiring)

**목적:** 신규 E-9 채용 요청의 준비 상태를 단계별로 추적한다.

**요구 화면 요소**
- 채용 요청 카드 목록 (진행도 표시)
  - 요청 ID, 공장/라인, 인원수, 준비도 퍼센트(%)
  - 완료/전체 태스크 수 (예: 5/8)
  - 남은 태스크 체크리스트
  - 상태 뱃지: `in_progress`, `review_required`
  - 마감일
- 신규 채용 요청서 초안 생성 버튼 → **승인 필요** 표시
- 후보자 패키지 연결 (`Candidate A` 등)

**UX 주의사항**
- 후보자 성실도 판단 UI 요소 금지
- 국적별 선호 표시 금지
- 모든 채용 확정 액션은 `승인 필요` 게이트 통과 필요

---

### 4-3. 근로자 (Workers)

**목적:** 현재 재직 중인 외국인 근로자 전체 현황을 확인하고 개인별 상세를 조회한다.

**목록 화면**
- 검색 바
- 근로자 카드 또는 테이블
  - 표시명 (PII 마스킹 처리됨 — raw 이름 ❌)
  - 국적 + 국기
  - 체류자격 + 만료일 D-day
  - 현재 케이스 수 / 위험도
  - 서류 상태 요약

**상세 패널 (슬라이드 오버)**
- 체류 정보 (만료일, 자격, 갱신 이력)
- 계약 정보 (시작일, 종료일, 사업장/라인)
- 서류 체크리스트 (필수 / 선택 / 누락)
- 연결된 케이스 목록
- 컨택 이력 (메시지 발송 기록)

**UX 주의사항**
- 외국인등록번호 원문 표시 금지 (`***-*******` 마스킹)
- 이탈 예측 / 성실도 점수 UI 요소 금지

---

### 4-4. 컨택 (Contacts)

**목적:** 외국인 근로자에게 다국어 메시지 초안을 생성하고 담당자가 승인 후 발송한다. 수신된 답변을 해석하고 상태를 업데이트한다.

**컨택 스레드 목록**
- 근로자별 스레드 카드
  - 표시명 + 국기
  - 언어 뱃지 (vi, id 등)
  - 채널 (Zalo, 카카오톡, 문자)
  - 상태 뱃지: `draft` / `pending_approval` / `sent` / `replied`
  - 최근 메시지 미리보기
  - 마지막 업데이트 시각

**메시지 초안 상세 패널**
- 한국어 원문 표시
- 번역본 (베트남어 등) 나란히 표시
- 개인정보 수집 목적 / 보관 기간 포함 여부 체크
- 제출 기한 포함 여부 체크
- **승인 버튼** (담당자 승인 전까지 발송 불가)
- 수정 요청 버튼

**응답 해석 패널**
- 근로자 원문 답변 표시 (베트남어 등)
- 한국어 번역 요약
- 서류 수신 여부 자동 추출
- 상태 업데이트 후보 (is_final=false 명시)
- 담당자 확인 필요 뱃지

**응답 시나리오 3종**
- 긍정 응답: 서류 수신 후 행정사 패키지 반영 안내
- 추가 정보 요청: 서류 형식 기준 재안내
- 응답 지연: 리마인드 메시지 제안 (D+2)

---

### 4-5. 케이스 (Visa/Cases)

**목적:** 체류·서류·계약 관련 리스크 케이스를 위험도별로 관리한다.

**케이스 목록**
- 위험도 필터: CRITICAL / HIGH / MEDIUM / LOW / INFO
- 케이스 유형 필터: 체류만료 / 서류누락 / 계약종료 / 고용변동

**케이스 카드**
- 케이스 ID
- 근로자 표시명 + 국적
- 케이스 유형 + 위험도 배지
- D-day 카운트다운
- 관련 법령 citation 배지 (등급: A/B/C)
- 추천 다음 행동
- 마지막 처리 이력

**케이스 상세**
- 타임라인 (상태 변화 이력)
- RAG 검색 근거 자료 목록 (출처, 제목, 스니펫, 갱신일)
- 연결된 서류 체크리스트
- 연결된 컨택 스레드

---

### 4-6. 행정사 검토 (Admin Package / Documents)

**목적:** 행정사·노무사에게 넘길 검토 패키지 초안을 생성하고 담당자 승인을 거쳐 전달 준비한다.

**패키지 목록**
- 케이스별 패키지 카드
  - 케이스 ID + 근로자 표시명
  - 생성 시각
  - 수신자 (행정사 사무소명)
  - 상태: `draft` / `pending_approval` / `ready`

**패키지 상세 (섹션 구성)**

| 섹션 | 내용 |
|------|------|
| 근로자 기본 정보 | 마스킹 처리 (외국인등록번호 `***-*******`) |
| 체류/계약 상태 | 체류만료일, 계약종료일 |
| 제출 서류 현황 | 여권사본, 외국인등록증, 근로계약서 보유 여부 |
| 근거 자료 | 법령명 + 조항 + 요약 |

**승인 게이트**
- 패키지 전달 버튼은 담당자 승인 전 비활성화
- 승인 완료 후 `전달 준비 완료` 상태로 변경

---

### 4-7. 판단 기록 (Evidence Log)

**목적:** AI 판단의 모든 근거와 승인 이력을 감사 가능한 형태로 보관한다.

**이벤트 목록**
- 이벤트 유형 필터: `risk_flagged` / `citation_resolved` / `briefing_emitted` / `approval_granted` / `message_drafted`
- 각 이벤트 행: 타임스탬프, 유형, 액터(system/user), 케이스 ID, 요약

**이벤트 상세**
- 입력 스냅샷 (input_snapshot)
- 출력 결과 (output)
- 인용 근거 목록 (citations)
- 리스크 플래그 (risk_flags)
- 승인 여부 (approval_required)

**UX 주의사항**
- 근로자 메시지 전문, 외국인등록번호 원문 표시 금지
- 마스킹된 식별자만 표시

---

### 4-8. 승인 센터 (Approvals) — 모바일 핵심 화면

**목적:** 현장 관리자가 이동 중 모바일에서 빠르게 승인/거절 처리한다.

**승인 카드 목록 (카드 스와이프 또는 버튼)**
- 태스크 ID + 유형 뱃지 (visa_document_request 등)
- 근로자 표시명 + 국기
- D-day 카운트다운
- 요청 내용 요약 (1-2줄)
- **승인** / **거절** / **수정 요청** 버튼

**승인 처리 후**
- Evidence Log 자동 기록
- 관련 담당자 알림

## 5. 핵심 UX 패턴

### 5-1. Human Approval Gate (가장 중요한 UX 패턴)

모든 외부 발송/전달/제출 전에 반드시 담당자 승인 단계가 있다.

```
AI 초안 생성
  ↓
[초안 미리보기 + 근거 표시]
  ↓
담당자 확인 및 수정
  ↓
[승인 버튼]
  ↓
Evidence Log 기록
  ↓
실행 (발송/전달)
```

**UI 표현 방식**
- 승인이 필요한 버튼: 잠금 아이콘 또는 `승인 필요` 뱃지
- 초안 상태: 노랑 `pending_approval` 뱃지
- 승인 완료: 초록 `approved` 뱃지
- 자동 실행 없음 명시: "AI가 직접 발송하지 않습니다" 문구 표시

### 5-2. 위험도 색상 체계

| 위험도 | 색상 | 액션 |
|--------|------|------|
| CRITICAL | 빨강 `#EF4444` | 즉시 확인 필요 |
| HIGH | 주황 `#F97316` | 우선 확인 |
| MEDIUM | 파랑 `#3B82F6` | 확인 필요 |
| LOW | 회색 `#6B7280` | 참고 |
| INFO | 초록 `#10B981` | 정보 |

### 5-3. D-day 뱃지 표현

| 조건 | 표시 | 색상 |
|------|------|------|
| 만료됨 | `D+N` | 빨강 |
| D-0 | `D-day` | 빨강 |
| D-1 ~ D-30 | `D-N` | 주황 |
| D-31 ~ D-90 | `D-N` | 파랑 |
| D-91 이상 | `D-N` | 회색 |

### 5-4. 데모 데이터 Fallback

API 연결 실패 시 demo fixture로 화면을 표시하되, **"데모 데이터"** 배지를 UI 상단에 반드시 표시한다.

### 5-5. 근거 인용 (Citation) 표시 패턴

AI 추천/판단이 표시되는 모든 곳에 법령 근거를 인라인으로 표시한다.

```
[A] 출입국관리법 제25조 — 체류기간 연장허가
    "외국인이 체류기간을 초과하여 계속 체류하고자 할 때에는..."
    ↗ 국가법령정보센터 · 2025.11.02
```

등급: A (법령) / B (공식 안내) / C (내부 패턴)

## 6. PC vs 모바일 분기 전략

### PC (인사 담당자 주 사용)

- 좌측 사이드바 네비게이션 (아이콘 + 텍스트)
- 중앙 콘텐츠 영역 (테이블/카드)
- 우측 슬라이드 패널 (상세/패널)
