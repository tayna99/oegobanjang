## 3. 공통 UI 컴포넌트 (`features/pc/ui.tsx`)

### `cn()` 유틸

```ts
export function cn(...classes: Array<string | false | null | undefined>) {
  return classes.filter(Boolean).join(" ");
}
```

### `Button`

```tsx
// variant: "primary" | "secondary" | "ghost"  (기본값: "primary")
// primary  → styles.button + styles.buttonPrimary  (bg:#2563eb, color:#fff)
// secondary → styles.button  (bg:#fff, border:#e2e8f0)
// ghost    → styles.button + styles.buttonGhost  (투명)
```

```css
/* 실제 스타일 */
.button {
  display: inline-flex; align-items: center; gap: 8px;
  min-height: 40px; border: 1px solid #e2e8f0; border-radius: 12px;
  padding: 0 16px; background: #fff; font-weight: 800; font-size: 14px;
}
.buttonPrimary { border-color: #2563eb; background: #2563eb; color: #fff; }
.buttonGhost   { border-color: transparent; background: transparent; color: #475569; }
```

## 9. 스타일링 실제 규칙

현재 코드에서 사용되는 스타일 방식은 딱 두 가지입니다.

**1. CSS Modules (PcShell.module.css) — 메인**

모든 PC 뷰의 레이아웃, 색상, 간격은 `styles.xxx`로 처리.  
`import styles from "./PcShell.module.css"`를 통해 사용.

```tsx
<div className={cn(styles.card, styles.briefing, className)}>
```

**2. Tailwind — globals.css에 `@tailwind` import만 있음**

`tailwind.config.js`에서 `content`에 `features/**/*.{ts,tsx}` 포함되어 있지만, 현재 views.tsx에서 실제 Tailwind 클래스를 사용하는 코드는 없습니다. 향후 확장용으로 준비된 상태.

> 신규 컴포넌트 작성 시: CSS Modules 방식으로 작성하거나, 별도 `.module.css` 파일 생성 후 import. 뷰에서 `className="text-sm"` 같은 Tailwind 클래스를 직접 쓰지 않습니다.
