# 외고반장 역할별 화면 기획 정리

작성일: 2026-05-08  
문서 목적: 외고반장의 최종 MVP 화면 방향을 **사장님용 모바일 채팅**과 **인력/행정 담당자용 PC 대시보드**로 분리하여 정리한다. 이 문서는 UI/UX 목업 제작 시 기준 문서로 사용한다.

---

## 1. 핵심 방향

외고반장은 하나의 화면을 모두가 같이 쓰는 서비스가 아니다.  
사용자 역할에 따라 화면의 목적과 형태가 달라져야 한다.

가장 중요한 구조는 다음과 같다.

> **대시보드는 인력 담당자의 작업대이고, 모바일 채팅은 사장님의 보고 채널이다.**

따라서 외고반장은 단순히 “대시보드 안에 AI 보조 패널이 있는 서비스”가 아니라,

> **PC 운영 콘솔 + 모바일 AI 반장 브리핑이 결합된 외국인 고용 운영 SaaS**

로 기획해야 한다.

---

## 2. 역할별 제품 구조

| 사용자 | 주요 화면 | 핵심 목적 | AI의 역할 | 화면 톤 |
|---|---|---|---|---|
| 사장님 | 모바일 채팅 | 오늘 뭐가 위험한지 빠르게 보고 큰 결정을 승인 | 오늘 브리핑, 위험 요약, 승인 요청, 질문 응답 | 친근하지만 단호한 AI 반장 |
| 인력 담당자 / 행정 담당자 | PC 대시보드 | 실제 케이스를 확인하고 서류·날짜·근거·승인 상태를 관리 | 리스크 계산, 서류/근거 정리, 액션 생성 | 차분한 실무 SaaS |
| 행정사 / 노무사 | Handoff Preview | 검토자료 확인 | 객관적 근거와 기초 자료 정리 | 문서형 / 근거형 |
| 관리자 / 감사자 | Evidence Log | 판단 이력과 승인 이력 확인 | 감사 가능한 기록 제공 | 감사 / 기록형 |

---

## 3. 사장님 화면

### 3.1 목적

사장님 화면의 목적은 다음이다.

> **오늘 뭐가 위험한지 빠르게 보고, 큰 결정을 승인한다.**

사장님은 테이블을 오래 보는 사람이 아니다.  
공장 운영, 사람 관리, 외부 미팅, 전화 대응을 하면서 빠르게 보고받는 사람이기 때문에 모바일 채팅이 적합하다.

사장님이 알고 싶은 질문은 다음과 같다.

- “오늘 큰일 날 거 있어?”
- “누가 곧 만료야?”
- “행정사한테 넘길 거 준비됐어?”
- “내가 승인해야 할 거 뭐야?”
- “이거 지금 처리 안 하면 문제 돼?”

따라서 사장님 화면은 **요약, 경고, 승인, 질문** 중심이어야 한다.

---

## 4. 인력 담당자 / 행정 담당자 화면

### 4.1 목적

인력 담당자와 행정 담당자 화면의 목적은 다음이다.

> **실제 케이스를 확인하고, 서류·날짜·근거·승인 상태를 관리한다.**

이 사람들은 매일 들어와서 실제 업무를 처리해야 하므로, PC 기반의 업무형 SaaS 대시보드가 적합하다.

필요한 정보는 다음이다.

- 근로자 목록
- 체류만료 D-day
- 계약 종료일
- 서류 누락 상태
- 승인 대기 액션
- 행정사 검토 패키지
- Evidence Log
- 업무 처리 이력
- 필터 / 정렬 / 상태 배지 / 테이블

즉, 인력 담당자 화면은 **조작 가능한 운영 콘솔**이어야 한다.

---

## 5. AI 반장의 위치

AI 반장은 대시보드 오른쪽 보조 패널에 있는 캐릭터가 아니다.

더 정확히는:

> **사장님 모바일 채팅 인터페이스 그 자체가 AI 반장이다.**

PC 대시보드 안에서는 AI 반장이 주인공이 되면 안 된다.  
인력 담당자 화면에서는 “AI”보다 **케이스, 날짜, 서류, 승인 상태**가 먼저 보여야 한다.

### 5.1 톤 비율 재정의

| 화면 | 권장 톤 |
|---|---|
| PC 대시보드 | 90% 실무형 SaaS |
| 모바일 채팅 | AI 반장 브랜드감이 살아나는 곳 |
| 캐릭터 사용 | 모바일 빈 상태, 안내, 브리핑에 제한적으로 사용 |

---

## 6. 인력 담당자용 PC 대시보드

### 6.1 화면 정체성

PC 대시보드는 인력 담당자의 작업대다.

대시보드의 주인공은 AI가 아니라 **업무 큐**다.

즉, 화면 중심은 다음이어야 한다.

> **오늘 처리할 외국인 고용 업무**

---

### 6.2 상단 영역

상단에는 업무 맥락과 탐색 기능이 들어가야 한다.

필수 요소:

- 사업장 선택
- 공장 / 라인 필터
- 오늘 날짜
- 브리핑 생성 / 새로고침
- 승인 대기 건수
- 검색창

“AI 반장에게 물어보기”가 들어간다면 메인 기능이 아니라 검색창 옆의 작은 보조 기능 정도로 제한한다.

---

### 6.3 오늘 확인할 일 카드

대시보드 상단에는 오늘 처리해야 할 일을 요약 카드로 보여준다.

예시:

- 체류기간 임박 4명
- 서류 보완 필요 7건
- 계약 종료 임박 3명
- 승인 대기 5건
- 행정사 검토 준비 2건

각 카드는 클릭 시 해당 필터가 적용된 업무 테이블로 연결되어야 한다.

---

### 6.4 업무 테이블

업무 테이블은 PC 대시보드의 핵심이다.

필수 컬럼:

| 컬럼 | 설명 |
|---|---|
| 근로자명 | 마스킹 또는 일부 표시 가능 |
| 국적 | 운영 정보로만 표시, 추천/우선순위에 사용하지 않음 |
| 사업장/라인 | 어느 공장/라인 소속인지 |
| 체류자격 | 예: E-9 |
| 체류만료일 | 날짜 표시 |
| D-day | D-30, D-45 등 |
| 계약 종료일 | 계약 만료일 |
| 서류 상태 | 누락/보완 필요/검토 완료 |
| 위험도 | 즉시 확인/우선 확인/확인 필요/참고 |
| 승인 상태 | 승인 대기/승인 완료/검토 필요 |
| 다음 처리 | 서류 요청 초안, 검토 자료 만들기 등 |

---

### 6.5 상세 패널 또는 상세 페이지

근로자를 클릭하면 상세 패널 또는 상세 페이지가 열려야 한다.

필수 요소:

- 기본 정보
- 체류 / 계약 상태
- 누락서류
- 근거 자료
- 추천 액션
- 승인 요청 상태
- 행정사 검토용 초안
- 업무 기록

## 7. 사장님 모바일 채팅

### 7.1 화면 정체성

사장님 모바일 채팅은 인력 담당자용 대시보드가 아니다.

이 화면은:

> **사장님이 이동 중에 보고받는 보고 채널**

이다.

따라서 모바일 채팅은 다음 역할을 해야 한다.

- 오늘/이번 달 뭐가 위험한지 알려준다.
- 누구에게 연락해야 하는지 짚어준다.
- 서류 요청 문구나 검토 자료 초안을 보여준다.
- 사장님 승인 전에는 아무것도 외부로 나가지 않는다.

---

### 7.2 참고해야 할 PDF 방향

모바일 채팅 화면은 PDF의 다음 흐름을 참고한다.

- **19페이지: 이번 달 처리 필요 업무 브리핑**  
  사용자가 “이번 달 외국인 직원 중에 처리할 거 있어?”라고 묻고, 외고반장이 Nguyen, Tran, 신규 채용 요청 같은 케이스를 요약하는 구조를 참고한다.

- **20페이지: 외고반장이 사장님께 드리는 5가지 가치**  
  무엇을 놓쳤는지 즉시 알 수 있음, 누구에게 연락해야 하는지 짚어줌, 외국어 메시지 자동 생성, 행정사 자료 정리, 사장님 승인 하 실행이라는 가치를 UX 원칙으로 삼는다.

---

### 7.3 모바일 첫 화면 구조

첫 화면은 단순 채팅창이 아니라 **오늘 브리핑 카드**가 먼저 보여야 한다.

상단 요소:

- 외고반장 로고
- 사업장명
- 오늘 날짜
- 알림 상태

첫 메시지 예시:

> 사장님, 이번 달 처리할 외국인 고용 업무가 4건 있습니다.

요약 카드:

- 즉시 확인 1건
- 우선 확인 3건
- 승인 대기 2건
- 검토 자료 준비 1건

---

### 7.4 채팅형 리스트 예시

#### 1. Nguyen V. — 체류기간 D-30

- 서류 보완 필요: 여권 사본, 외국인등록증 사본
- 상태: 우선 확인
- 다음 처리: 서류 요청 초안 확인

#### 2. Tran T. — 계약 종료 D-45

- 체류기간과 계약 종료일 확인 필요
- 상태: 검토 필요
- 다음 처리: 담당자 확인 요청

#### 3. 신규 채용 요청 3명

- 준비 상태 확인 중
- 상태: 준비 중
- 다음 처리: 요청서 초안 보기

---

### 7.5 모바일 버튼

모바일 채팅 하단에는 빠른 액션 버튼을 둔다.

권장 버튼:

- 서류 요청 초안 보기
- 검토 자료 만들기
- 담당자에게 확인 요청
- 승인 대기만 보기
- 나중에 보기

## 8. 모바일 채팅에서 안전하게 바꿔야 할 용어

PDF 19페이지에는 “서류 요청 메시지 발송”, “행정사 패키지 생성” 같은 버튼 표현이 나온다.  
하지만 최종 MVP에서는 자동 실행처럼 보일 수 있으므로 더 안전한 표현으로 바꿔야 한다.

| 위험하거나 과한 표현 | 안전한 MVP 표현 |
|---|---|
| 서류 요청 메시지 발송 | 서류 요청 초안 보기 |
| 행정사 패키지 생성 | 검토 자료 만들기 |
| 행정사에게 전달 | 행정사 검토용 자료 준비 |
| 실행 완료 | 승인 후 처리 가능 |
| 비자 가능 | 전문가 검토 필요 |
| 채용 가능 | 준비 상태 확인 |
| 자동 처리 완료 | 승인 대기 |

핵심 원칙:

> **발송 / 전달 / 확정 / 완료보다 초안 / 검토 / 준비 / 승인 대기라는 표현을 우선 사용한다.**

---

## 9. 모바일 채팅에서 꼭 들어가야 하는 4개 화면

### 9.1 오늘 / 이번 달 브리핑 화면

사장님이 앱을 열자마자 보는 화면이다.

문구 예시:

> 사장님, 오늘 먼저 확인할 업무입니다.  
> 즉시 확인 1건, 우선 확인 3건, 승인 대기 2건이 있습니다.

필수 요소:

- 요약 카드
- 위험 건수
- 승인 대기 건수
- 검토 자료 준비 건수
- 케이스별 간단 요약

---

### 9.2 케이스 상세 채팅 화면

Nguyen 케이스를 누르면 상세 채팅이 열린다.

문구 예시:

> Nguyen V.의 체류기간이 30일 남았습니다.  
> 현재 여권 사본과 외국인등록증 사본이 보완 필요 상태입니다.  
> 담당자가 서류 요청 초안을 준비할 수 있습니다.

하단 버튼:

- 서류 요청 초안 보기
- 검토 자료 만들기
- 근거 확인
- 담당자에게 맡기기

---

### 9.3 서류 요청 초안 화면

다국어 컨택은 한국어 요청문과 외국어 요청문을 함께 보여주는 구조가 좋다.

필수 요소:

- 한국어 요청문
- 베트남어 등 외국어 요청문
- 서류 요청 이유
- 제출 기한
- 개인정보 사용 목적 안내
- 담당자 승인 상태

버튼:

- 초안 확인
- 수정 요청
- 담당자에게 전달
- 승인하기

주의:

> 실제 발송은 Sprint 1 MVP 범위 밖이다.  
> 이 화면은 발송 화면이 아니라 초안 검토 화면이다.

---

### 9.4 승인 전 확인 화면

승인 전 확인 화면은 모바일 채팅에서 가장 중요한 안전 장치다.

필수 문구:

> 승인 전에는 외부로 발송되지 않습니다.  
> 이 승인은 내부 처리 상태를 ‘승인됨’으로 바꾸는 단계입니다.  
> 실제 발송/전달은 담당자 확인 후 진행됩니다.

필수 요소:

- 승인 대상 액션
- 관련 근로자
- 관련 케이스
- 검토 자료 요약
- 근거 보기
- 승인하기
- 나중에 보기

---

## 10. 최종 화면 세트

최종 목업 구성은 4개로 잡는다.

---

### 10.1 인력 담당자 PC 대시보드

목적:

> 오늘 처리할 업무를 확인하고 관리한다.

필수 요소:

- 오늘 할 일 카드
- 근로자 리스크 테이블
- D-day 필터
- 서류 상태 필터
- 승인 상태 필터
- 상세 패널

### 10.2 근로자 / 케이스 상세 페이지

목적:

> 한 케이스를 실제로 처리한다.

필수 요소:

- 근로자 정보
- 체류만료
- 계약 종료
- 누락서류
- 추천 액션
- 승인 상태
- 근거 자료
- 업무 이력

---

### 10.3 사장님 모바일 채팅

목적:

> 요약 보고와 승인 중심으로 빠르게 확인한다.

필수 요소:

- 오늘 브리핑
- 위험 요약 카드
- 승인 대기 카드
- 채팅 질문

## 13. 최종 정리

외고반장은 역할별로 화면이 달라져야 한다.

- **사장님은 모바일 채팅으로 보고받고 승인한다.**
- **인력 담당자는 PC 대시보드에서 실제 업무를 처리한다.**
- **행정사/노무사는 Handoff Preview로 검토 자료를 확인한다.**
- **관리자/감사자는 Evidence Log로 판단 이력을 확인한다.**

최종 한 줄 정의:

> **외고반장은 PC 운영 콘솔과 모바일 AI 반장 브리핑이 결합된 외국인 고용 운영 SaaS다.**

목업 제작 시 가장 중요한 판단 기준:

> **PC는 업무 큐 중심, 모바일은 사장님 보고 중심, AI 캐릭터는 모바일 브리핑에서만 제한적으로 사용한다.**
